# Безопасность

Обновлено: 18.07.2026

## Обязательный system-wide security и privacy review

Каждое крупное cross-system изменение проверяет единообразие authorization, IDOR, CSRF, mass assignment, XSS, SSRF, path traversal, open redirects, uploads/private files, token/session handling, cache leakage, service-worker caching, payment/webhook integrity, advertiser privacy, legal-case privacy, ticket privacy, account merge/deletion, audit secrecy, export protection и administration least privilege. Решение остаётся server-side; browser-supplied permission, ownership, premium, region, price, status или legal state никогда не является authority.

Пункт считается исправленным только после конкретного code/configuration change и доступной проверки. Непроверенные provider, production или credential-dependent boundaries отмечаются `unresolved`/`not_performed`, а не «secure».

Task 27 verified account lifecycle against every SQLite FK to `users`. Единственный `RESTRICT` — immutable `admin_audit_events.actor_id`; `AccountService` теперь проверяет его до cleanup и возвращает локализованную retention validation вместо SQL error/partial deletion. Generic morph notifications, не защищённые FK, удаляются явно. Export выдаёт database notification data только по stable type/field allowlist и не включает private notes, message body, raw URL или неизвестные future fields.

## Security requirements для upgrades

- Advisories оцениваются только verified package tooling или authoritative source; package не называется compromised/vulnerable без evidence применимости к installed version.
- Updates не ослабляют CSRF, sessions, cookies, OAuth, webhooks, uploads, storage, authorization, CSP/CORS, SSRF, redirects, payment integrity и secret redaction.
- Package auto-discovery, service providers, middleware, routes, commands, scheduler tasks, queue jobs, public assets и browser permissions проходят отдельный review.
- Новые environment variables документируются без значений; package removal включает secret/environment cleanup.
- Third-party telemetry запрещена без explicit approval. Analytics/error/usage packages проверяются на передачу private data.
- Abandoned security-sensitive package получает evidence-backed staged replacement plan.
- Source maps, debug packages, toolbars, profilers и local diagnostics не включаются в production случайно.
- Broad CORS, debug output и new public endpoints не принимаются как package defaults без authorization/privacy review.

## Production secrets, backups и diagnostics

- Secrets остаются вне Git, administration, health payloads, screenshots и документации; diagnostics показывают только configured/missing/reachable state с redaction.
- Production требует `APP_DEBUG=false`, safe error pages и secure cookies за HTTPS. Trusted proxy/CORS boundaries не расширяются без проверки фактического reverse proxy.
- Logs не содержат passwords, tokens, authorization headers, cookies/session IDs, payment/private-provider data, protected source URLs и legal/private attachment paths or contents.
- Public upload storage не исполняет scripts; private storage и backups не публикуются web server. Database dumps и backup archives никогда не хранятся под public web root.
- Backup retention и restore authorization документируются; restore/emergency admin требуют отдельного least-privilege permission, recent authentication и audit.
- Webhook signatures проверяются server-side; production callback URLs используют HTTPS там, где это требует provider. Health endpoints не раскрывают packages, hostnames, database names, usernames, paths, exceptions или stack traces.
- Log/file viewers используют allowlisted targets и защищены от path traversal; incident exports остаются private. Emergency access отзывается после инцидента.

## Правила

- `.env`, ключи, токены, cookies, приватные логи и локальные базы не коммитятся; публично отслеживается только `.env.example` без значений секретов.
- Mobile personal access tokens хранит Laravel Sanctum только как SHA-256 hashes; expiry ограничен 129600 минутами, а ежедневный prune удаляет просроченные строки. Token plaintext не логируется и после issuance/rotation не возвращается повторно.
- Register/login не различают неизвестный аккаунт и неверный пароль; forgot-password возвращает одинаковое тело для существующего и отсутствующего email. Verification/reset links создают только project-owned queued notifications. Password reset отзывает все mobile tokens; authenticated смена пароля, смена email, повторная регистрация адреса и удаление аккаунта также удаляют применимые reset rows.
- Все `/api/*` ошибки используют стабильный русскоязычный envelope и request ULID без exception message/stack trace. Ответы с `Authorization`, authenticated user, cookie, error или unsafe method принудительно получают `private, no-store` без shared validators.
- OAuth client secrets, Google ADC/service-account JSON, refresh tokens, MCP bearer tokens и приватные пути к credential-файлам не хранятся в репозитории.
- Код приложения читает переменные окружения через `config()`. Прямые `env()` допустимы только в `config/*.php`.
- Публичные страницы каталога остаются read-only и проходят строгую валидацию query-параметров через Form Request-классы.
- Все текущие публичные маршруты с implicit binding `CatalogTitle`, включая карточку `/titles/{slug}`, API show и proxy постера статистики, доступны только при `is_published=true`; неопубликованный тайтл возвращает `404` до controller/responder и не инициирует внешний HTTP-запрос.
- Служебная страница `/stats` доступна гостям как read-only Livewire-сводка и не выводит raw source URLs, приватные media URLs, stack traces или внутренние имена маршрутов.
- Livewire-компонент `/stats` получает данные через очищенный snapshot: полный массив статистики не хранится в публичных свойствах компонента и не должен попадать в `wire:snapshot`.
- Livewire-компонент карточки хранит только locked `catalogTitleId` и bounded URL-скаляры. Watchlist/rating/progress actions требуют authenticated user, а `CatalogUserStateService` сам применяет `CatalogTitlePolicy::interact` и повторно разрешает episode/media внутри доступной hierarchy; пользовательские/profile IDs не принимаются аргументами записи. Список просмотра принимает только желаемый boolean, а оценка — только значение из серверного диапазона.
- Общие локальные HTTP- и action-level rate limits публичного каталога отключены по решению владельца проекта. Исключение — узкие named budgets mobile credential endpoints: register/login, verification resend, password recovery и token refresh. Authentication, authorization, CSRF, signed playback URL, validation, URL allowlists и bounded importer retries остаются обязательными границами. HTTP 429 от Seasonvar/CDN считается внешней transient-ошибкой и не является ограничением входящих запросов приложения.
- Playback source выдается только через подписанный на пять минут `/playback/{licensedMedia}` с viewer binding. Resolver получает `CatalogEntitlementDecision` для каждого parent/media, повторно проверяет принадлежность media выбранной серии, известные source failures, формат и storage/host allowlist. Raw provider URL, private path и credentials не сериализуются в Livewire и не принимаются от браузера.
- Mobile playback использует отдельный same-origin signed `/api/v1/playback/{licensedMedia}` и encrypted grant с version, nullable user ID, media ID и expiry. JSON session возвращает только безопасный media profile и opaque URL; raw provider URL, `path`, source metadata и storage credentials не попадают в JSON, exception body или логи. Delivery повторяет user/media/hierarchy entitlement check непосредственно перед redirect и закрывается `private, no-store`, `no-referrer` и `nosniff`.
- Bearer token никогда не помещается в playback query string. Progress token отделён от delivery grant, выдаётся только verified пользователю, привязан к user/title/episode/media и защищён TTL с монотонным event sequence; response не возвращает session ID, grant или media source.
- Offline-sync cursor подписан и привязан к public/private scope и owner; он не содержит token, credential или сериализованную model. Private pull/push всегда `private, no-store`. Подменённый scope/owner отклоняется до query, а устаревший cursor требует полный bootstrap вместо чтения уже удалённой истории.
- `api_sync_changes` хранит только type/key/operation/time invalidations. Idempotency receipt хранит SHA-256 canonical payload и безопасный result, но не request body, email, Bearer token, playback session/grant, media/source URL, importer state или exception. Одинаковый UUID с другим payload не выполняется; ожидаемые operation-level отказы очищены до стабильных codes.
- Livewire update и temporary upload endpoints используют стандартный `web` middleware stack без локального request budget; encrypted cookies, CSRF и Livewire signature остаются обязательными.
- Video sitemap использует только внутренний `video:player_loc` страницы тайтла; постоянный provider URL и storage path не публикуются как `video:content_loc`.
- Внешняя playback-проверка допускает только HTTPS:443 hosts из `PLAYBACK_ALLOWED_HOSTS`, отклоняет credentials, localhost, private/reserved/link-local/metadata IP и любой host с непубличным A/AAAA. Выбранный проверенный адрес закрепляется через cURL resolve на один запрос, поэтому повторное DNS-разрешение HTTP-клиентом не открывает SSRF; redirects отключены.
- Для обычного видео отправляется bounded Range GET, для HLS читается только ограниченный manifest fragment и проверяется `#EXTM3U`. `SEASONVAR_MEDIA_CHECK_MAX_RESPONSE_BYTES`, connect/total timeout и закрытие stream не позволяют скачать полный файл. В события попадают только HTTP status, latency и allowlisted error category; URL, query tokens, authorization headers, response body и exception message не сохраняются.
- Прокси `stats.poster` всегда делает запросы только к HTTPS-URL с валидацией `poster_url`, закрепляет проверенный публичный A/AAAA через cURL resolve, не следует редиректам (`withoutRedirecting()`), отбрасывает слишком большие изображения и проверяет `Content-Type` как `image/*`. Непустой `Content-Length` проверяется до чтения тела, а фактический размер тела проверяется всегда; корректные chunked-ответы без `Content-Length` допускаются.
- `CatalogStatsPosterUrlGuard` используется и при сборке `/stats`, и в proxy responder: неразрешимые hostnames, localhost, `.local`, private и reserved IP не попадают в `poster_src` и не создают браузерные 404-запросы.
- Внешние URL Seasonvar нормализуются и допускаются только для `seasonvar.ru`; внешние playlist URL не могут содержать credentials, указывать на localhost, `.local`, private/reserved A/AAAA или проходить через redirect. Публичный адрес входного playlist закрепляется на запрос.
- Google service-account `token_uri` принимается только в каноническом виде `https://oauth2.googleapis.com/token`, поэтому подписанный JWT assertion нельзя перенаправить на endpoint из подмененного credential JSON.
- Google service-account access token не сохраняется в Redis, Memcached, session или другом cross-request cache; он существует только в памяти текущего API-вызова. Cache layer не хранит passwords, CSRF, raw tokens, credential paths, raw signed media URL или private authorization state.
- DRM/license exchange, хранение provider credentials и browser delivery таких credentials не реализованы и не являются незавершённой security-задачей текущего продукта. Приложение хранит только проверенные внешние playback URL; добавление DRM требует отдельного provider contract, secret storage, license proxy boundary и threat model до изменения кода.
- Public cache dimensions валидируются, ограничиваются и canonical-hash-ируются; raw search, IP, user ID и token не становятся ключом или metric label. Shared snapshots имеют explicit public audience/locale, а authenticated/non-default facets bypass-ят public tier.
- `/health/ready` не создаёт session, имеет `no-store, private`, `noindex,nofollow` и возвращает только общий `status`, `ready` и `checked_at`. Component names/latency/metrics, Redis/Memcached hostnames, queue state, DB paths и credentials остаются только в operator CLI diagnostics.
- Внешние MCP и Google Workspace данные считаются недоверенным контентом; write tools и широкие scopes включаются только под конкретную задачу и с user-level authorization.
- Локальные temporary storage URLs отключены по умолчанию через `LOCAL_FILESYSTEM_SERVE=false`; включать их можно только для явной функции загрузки/выдачи файлов с отдельной авторизацией.
- Пользовательские uploads по умолчанию сохраняются на приватный disk `uploads`; нельзя доверять клиентским именам файлов, делать upload-файлы публичными без отдельной авторизации или отдавать private paths наружу.
- Operational logs/notifications не включают targeted importer URL, stack traces, raw HTML source snapshots, private media URLs, query tokens, секреты или credential paths; сохраняются режим, run/source IDs, класс исключения и очищенная категория/ошибка.
- `/admin/catalog` закрыт тем же configured email allowlist, что importer admin, и дополнительно использует policy, hierarchy-scoped lookup и locked IDs/versions. Stored playback URL не рендерится; новый URL проходит общий SSRF/provider allowlist.
- Blade-шаблоны не содержат `@php`/`@endphp`; вывод экранируется через `{{ }}`, кроме JSON-LD с `JSON_HEX_*` флагами.

## HTTP

- Web-ответы добавляют защитные заголовки: `X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`, `Permissions-Policy` и `X-Permitted-Cross-Domain-Policies`.
- HTML-ответы дополнительно получают `Content-Security-Policy-Report-Only`. Fixed policy запрещает objects, ограничивает base/form/frame, не содержит `unsafe-eval`, а image/media/connect sources берёт только из очищенных `SECURITY_CSP_*_SOURCES`. Livewire 4 использует официальный CSP-safe bundle через `config/livewire.php`, поэтому каталог не требует добавлять `unsafe-eval`. JSON API не получает этот body-dependent header.
- CSP пока не enforced: текущий каталог использует несколько external poster/media origins, а inline styles нужны существующему UI/framework. Сначала нужно наблюдать violations, сузить `https:` до подтверждённых origins и только затем включать enforcement; публичный report collector намеренно не добавлен.
- Laravel web middleware сохраняет стандартные encrypted cookies и `PreventRequestForgery` для небезопасных HTTP-методов.

## Security boundary аутентификации

- Credential boundary использует только Laravel guard/provider/`Hash`/password broker. Email централизованно trim/lowercase-normalize-ится без provider-specific переписывания, password не trim-ится, paste/password managers разрешены, policy едина для register/reset/change. Registration принимает explicit name/email/password и не может mass-assign role, premium, verification или restriction state.
- Enumeration policy: login и recovery отвечают generic сообщениями; unknown mobile email выполняет dummy secure hash work; verification resend доступен только текущему unverified owner. Duplicate registration сообщает безопасный conflict с login/recovery path, не раскрывая profile/provider/private data. Matching social email никогда не link/merge-ит account, а social surface отсутствует до отдельного proof-of-control domain.
- Layered limiter keys используют HMAC application-secret fingerprints нормализованного email, network address и operation scope; raw email/IP/password не становятся cache keys или log context. Login одновременно ограничен pair bucket `5/minute`, normalized-identifier bucket `20/10 minutes` и network bucket `60/minute`, поэтому смена IP или email не обходит полную границу. Browser/API recovery использует pair/identifier/network budgets `3/3/30` за 10 минут, reset — `3/10/30`; успешный reset очищает только pair/identifier buckets, сохраняя network spray budget. Отдельные budgets также действуют для register, verification resend, password/email/session/device/deletion actions и mobile token issuance/rotation.
- Successful browser login регенерирует session ID; logout инвалидирует session и регенерирует CSRF token. State-changing web/Livewire actions остаются POST с CSRF. `Secure` определяется HTTPS deployment, authentication cookie всегда HttpOnly, `SameSite=lax`, path/domain не расширяются. Remember token остаётся server-owned Laravel value и вращается при reset/change/logout-all policy. Password/email/deletion mutations внутри своих транзакций повторно получают user row с `lockForUpdate`, проверяют current password по свежему hash и только затем меняют credential state или удаляют account; stale запрос со старым паролем не может перезаписать новый hash, сменить login email или удалить аккаунт.
- `AuthenticationRedirectService` допускает только validated relative/same-origin paths, удаляет auth callback/logout/reset loops и проверяет security path после двух URL-decode проходов. Scheme/credentials/foreign host/port, protocol-relative/repeated slash, dot-segment, malformed percent encoding, backslash, control, fragment и encoded auth-route обходы отклоняются. Intended destination хранится в session, не global cache, query token или analytics.
- Verification URL временно signed и дополнительно сверяет user ID/email hash; signed-locale middleware принимает только `ru|en` до signature response. Password broker storage/expiration/single-use semantics остаются Laravel-owned. Notification URLs не логируются, а reset/verification tokens отсутствуют в canonical, Open Graph, JSON-LD и sitemap.
- Database sessions не загружают payload; browser action использует HMAC opaque token и повторно проверяет owner/current-session rules. Sanctum stores token hash only and resolves device ID through owner relation. Refresh повторно owner-resolves persisted token под transaction `lockForUpdate`, удаляет его до replacement issuance и отвечает generic `unauthenticated`, если concurrent/stale request уже потребил строку; один исходный token поэтому создаёт не более одного активного replacement. Password reset/delete revoke applicable tokens; `auth.session` invalidates browser sessions after password hash changes.
- Security audit uses stable allowlisted event enum plus HMAC email/network fingerprints. It never writes password/hash, raw email/IP/UA, cookies, session/token IDs, reset/verification/OAuth codes, provider tokens, request body or exception message. Dedicated daily retention is configurable by `AUTH_AUDIT_DAYS`; log access follows operations policy, not normal admin UI.
- Authenticated/private responses are `private, no-store`; identity, verification, restrictions, intended URL and secrets never enter shared response/data caches. Cache/config/log failure does not relax authentication or authorize access. Account export excludes security secrets; deletion calls the existing transaction, clears reset tokens/tokens/sessions through canonical lifecycle and never deletes unrelated catalog records.
- Social providers/external identities, PKCE/OAuth state, automatic account merge, magic link, MFA, trusted-device and account-status stores are absent. No dead button or partial schema is exposed. Adding them requires approved provider dependencies, encrypted secret storage, stable provider subject uniqueness, OAuth state/PKCE, recent authentication and explicit proof of both accounts; email equality alone is forbidden.

## Проверки

- `SecurityHardeningTest`, `LocalRateLimitRemovalTest`, mobile playback tests и существующие importer maintenance tests проверяют security headers и CSP source normalization, отсутствие локального HTTP 429 на публичных маршрутах, отключенные storage routes, entitlement/source recheck, opaque grant tampering/expiry, deleted-user/revoked-access boundary, playback fallback, blocked private/local/metadata hosts, unsafe redirects, timeout classification, redacted logs, failure thresholds, permanent failures и recovery.
- Mobile auth privacy matrix проверяет guest/invalid-token, unverified/verified, missing ability и cross-user device ID: каждая ошибка имеет request ID, `private, no-store` и не содержит submitted secret, token hash, SQL или stack trace. `LocalRateLimitRemovalTest` отдельно фиксирует отсутствие budgets на public routes и точный named limiter на каждом credential route.
- Offline-sync privacy tests проверяют tampered/cross-owner cursor, exact batch shapes, UUID replay/collision, optimistic conflicts, owner isolation, отсутствие grant/raw media/source/token в response и receipt, безопасный `sync_unavailable` до migration и bounded retention.
- Датированный результат review, accepted/deferred risks и внешние источники находятся в `docs/audits/security-audit.md` и `docs/research/laravel-video-portal-sources.md`.
- `PrivateImageUploadRulesTest` и `PrivateUploadStorageTest` проверяют upload-валидацию, private visibility, generated filenames и cleanup через fake storage.
- `RunSeasonvarImportJobTest` и `SeasonvarImportFailedNotificationTest` проверяют dispatch/content operational notification без отправки реальных писем.
- `CatalogPageTest` и `CatalogStatsSnapshotSanitizerTest` проверяют, что stats HTML/Livewire-рендер не раскрывает исходные source/media URL, stack traces и приватный event context.
- `composer audit` и `npm audit --audit-level=high` используются для проверки известных уязвимостей зависимостей.

## Security boundary коллекций

- IDOR: public URL использует slug→stable record, management/membership — opaque UUID; every view/mutation/cover/legacy redirect re-resolves and authorizes. Unauthorized private/pending/deleted targets return safe 404 before metadata/redirect.
- Input/XSS: enum/type/sort/report values are allowlisted, reorder is unique/bounded and collection-scoped, title identity is resolved server-side. `UserPlainText` normalizes Unicode and removes HTML/script/style/control/bidi content; Blade escapes it and editorial SEO uses the same plain-text boundary.
- Upload: reusable private-image rules and `PrivateUploadStorage` enforce raster MIME/size/generated paths. Delivery requires configured disk, exact UUID directory, current version, an explicit JPEG/PNG/WebP MIME, an existing file, no dot/backslash traversal, `nosniff`, noindex and no-store; missing/anomalous metadata fails as `404`, and executable/HTML content types are never served. Replaced files are removed only after the DB operation succeeds. Post-commit orphan cleanup is best-effort and safely reported, so an unavailable private storage backend cannot suppress the already-required cache/privacy transition or mask the original database failure.
- Privacy/cache: private membership, owner controls, report/moderation notes and personal state never enter public cache/API/structured data. Visibility/moderation changes bump every affected domain after commit; a fulfilled retry reissues derived invalidation if that post-commit step previously failed, while direct policy/query truth never trusts cache. Collection API/sitemap have zero shared/stale TTL and page/cover responses are no-store.
- Abuse/concurrency: create/mutate/cover/membership/reorder/report have user-scoped RateLimiter budgets. Stable UUID create tokens make retry idempotent; collection/cover/item mutations re-authorize locked current rows, add/batch revalidate current title entitlement, and DB locks, unique item/report keys, optimistic `content_version` plus transaction retries prevent duplicate submit/stale edit/order races. Report submission re-authorizes locked current visibility and atomically inserts its content-version key. Moderation/feature/report resolution re-authorize locked state, commit audit atomically, treat completed exact retries as no-op and bound bulk report closure to 100 rows per request.
- Destructive lifecycle: no GET mutation exists. Soft delete revokes access immediately; permanent delete is owner/admin or scheduled after the documented window, removes only owned cover/structural rows, preserves report evidence and privacy-retires generic comments without touching catalog titles or personal viewing state.

## Security boundary обсуждений

- IDOR/authorization: client передаёт только stable comment/target IDs и enum codes; каждое action повторно загружает row, применяет `CommentPolicy`, target allowlist и exact target/parent relation. Component target IDs, reply/edit/report mode IDs, edit version и submission tokens являются `#[Locked]`; изменяемыми остаются только валидируемые body/enum/form values. Create/edit/delete/restore/reaction/report transactions блокируют server-owned user, allowlisted visible target и comment в едином порядке, закрывая concurrent target retirement/orphan и account-deletion late-write races. Edit повторяет canonical anti-spam decision и переводит новый link-bearing account edit в `pending`, поэтому первоначальный publish без ссылки не обходит pre-moderation. Author, target, parent/root, status, moderator и user ID нельзя изменить mass assignment-ом. Все mutations — Livewire POST с CSRF; direct-comment GET только reauthorizes и redirects.
- XSS/link safety: `CommentBody`/`UserPlainText` NFKC-normalize plain text, удаляют tags/script/style/control/bidi и отклоняют dangerous schemes/excessive URL tokens. Blade использует escaped output, не `{!! !!}`; URL-like body не превращается в link/rich preview, поэтому `javascript:`, handler attributes, iframe, Blade/template fragments и provider HTML не исполняются.
- Replies: parent/reply target обязан совпадать; server вычисляет root и не предоставляет reparent API. Один structural level, FK и root checks исключают self/descendant cycle, cross-target reply и recursive render/query abuse. Deleted/non-published parent не принимает новый reply.
- Abuse: verified-email write boundary, per-action user/target RateLimiter buckets, UUID submission-key idempotency, 90-second same-user/target/root duplicate window over normalized case-insensitive body text, bounded length/lines/links/mentions/repetition и cautious pending signal. Weak signal не блокирует account и не отправляет UGC наружу.
- Privacy: spoiler/full long body отсутствует в unrevealed DTO/HTML, inbox, profile preview и metadata. Task 14 public-profile rows/count используют тот же published builder с exact title/season/episode availability и viewer block/mute context; `CommentPresenter` всегда даёт `null` spoiler excerpt, server-owned title/direct routes и не передаёт moderation, reaction, relationship или exact watch state profile layer. Поэтому снятая с публикации, region/premium-недоступная exact season/episode цель и muted author не раскрываются через public-profile count/excerpt. Non-public/deleted/blocked/muted tombstone не раскрывает author identity или relationship controls viewer-у без права читать body. Public count обсуждения не раскрывает blocks/mutes; block/mute sets, current reaction, pending ownership, restrictions, reports/reporter, notification state и moderation controls/notes не входят в shared cache.
- Reports/moderation: category/status/reason/restriction values enum-backed и не переводятся в DB. Detail/private note проходят plain-text limits. Reporter скрыт от public DTO; moderator fingerprints не содержат body/note. Deleted/hidden target остаётся только в gate-protected admin context.
- Account lifecycle: schema capability не зависит от public feature flag, поэтому cleanup выполняется и при `COMMENTS_ENABLED=false`. Export owner-only, password-confirmed, private и `no-store`; он исключает чужие data, moderator notes и report evidence. Общий account snapshot сейчас materialize-ится до streamed HTTP response, поэтому это не заявляется как memory-bounded row stream. Inbox mark-read принимает только UUID и повторно ограничивает lookup текущим recipient плюс stable notification type. Account row блокируется до domain cleanup и hard-delete; author/report linkage anonymize-ится, user-derived keys и private reactions/relationships/preferences/restrictions/notifications удаляются, thread integrity остаётся.
- Direct-link/SEO: localized closure объявляет `locale` и `comment` раздельно, responder повторно принимает только positive numeric identity, а parent хранит normalized focus как `#[Locked]`. Query-state title page получает clean canonical и `noindex,follow`; redirect остаётся `private, no-store`/`noindex,nofollow`. Невалидный, hidden или недоступный ID не может заставить eager island раскрыть body.

## Security boundary отзывов

- Rollout capability fail-closed проверяет не только identity/ownership, но и все lifecycle columns, которые реально используют edit/delete/moderation. Частично применённая schema не показывает writable controls и не доходит до SQL failure. Stable `removed` transition атомарно создаёт moderator tombstone; повторная модерация сохраняет первоначального deletion actor, а переход в другой status очищает только moderator evidence, не author/merge deletion.
- Target/IDOR: browser supplies stable numeric review/title IDs only. `ReviewTargetType::Title`, visibility resolver and `CatalogTitleReviewPolicy` reject arbitrary morph class, missing/hidden/deleted title, wrong owner, provider edit, merged/deleted review and forged moderation/verified state. Direct GET never mutates and only redirects after authorization; all writes use CSRF-protected Livewire requests.
- Text/XSS: `ReviewTitle`/`ReviewBody` reject unsupported mixed shapes and oversized raw byte payloads before `UserPlainText` NFKC/plain-text normalization, meaningful Unicode line preservation, control/bidi/HTML stripping and dangerous-scheme/link/line/repetition bounds. Blade uses escaped interpolation; HTML, Markdown, handlers, iframe, JavaScript/data/vbscript URL, Blade/template fragments and rich previews cannot execute. Provider text stays escaped and is never fed to a raw renderer.
- Abuse/concurrency: verified email, per-target plus global-account action budgets, UUID submission key, deterministic ownership key, short duplicate hash window and transaction locks make create/vote/report retry-safe. Guest spoiler reveal adds a hashed network-address budget without device fingerprinting. Vote uses unique-key upsert, report unresolved key uses insert-ignore. A bounded new-account/link signal may set pending but never automatically creates a permanent restriction or sends content to an external service.
- Spoiler/privacy: unrevealed spoiler title and body are absent from DTO/HTML/profile/notification/search/metadata/schema. The Task 14 public-profile tab obtains its typed item and count from the canonical review query/presenter rather than duplicating a body-only mask, preventing spoiler titles from appearing in cards. Its list and count share one bounded viewer relationship check, so muted/blocked authors cannot leak through profile activity or its counter. Exact watch progress, reporter, moderator note, block/mute sets, current vote, restriction and pending owner state never enter public API/cache. Public title aggregate does not vary by viewer relationship; viewer overlay stays private/no-store.
- Moderation: status/category/reason/vote/restriction values are stable enum codes, never translated DB labels. Report detail and private note have independent plain-text limits; audit stores safe fingerprints, not review body/note. Moderator routes/actions require `manage-reviews`, and target inactivity does not destroy evidence.
- Account/cache: account export is owner/password-confirmed and excludes reports/notes/exact evidence. Deletion anonymizes author/reporter and removes private engagement/preferences/notifications before user hard delete. Targeted after-commit invalidation prevents stale title/API/recommendation data; no cache key accepts raw sort/column/class input and no global user HTML is cached.

## Security boundary тегов

- Identity/IDOR: public global routes принимают только bounded slug и разрешают current/history/approved alias/merge через allowlisted models. Private API/Livewire принимают opaque UUID, но owner всегда current authenticated user и полный batch повторно сравнивается с owner-scoped active rows; client не передаёт owner/type/visibility/moderation/provider/model class. Unauthorized global/personal/title state отвечает 404/403 без canonical slug, name или count leakage.
- Authorization/CSRF: `TagPolicy`, `UserTagPolicy`, `CatalogTitlePolicy` и route gates применяются в каждом service/action и Livewire hydration. Every mutation — POST/PATCH/PUT/DELETE or Livewire CSRF transport; GET routes только читают/redirect. Global assign/merge/archive/provider moderation требуют `manage-catalog`; personal create/edit/restore/assign — verified owner.
- Text/XSS: `TagNormalizationService` и `PlainText` decode entities before stripping markup, reject control/format/invisible input, preserve meaningful Unicode, enforce length/letter-or-number content and never render provider HTML. Descriptions/SEO/aliases are escaped plain text; no raw HTML highlighting, template fragment or automatic translation exists. Narrow locale-aware reserved names prevent system impersonation without banning ordinary labels.
- Duplicate/graph safety: exact SHA-256 of normalized comparison is scoped globally or per owner; DB uniqueness and transaction locks prevent race duplicates. Fuzzy/transliteration/translated similarity never auto-merges. Alias maps to exactly one tag: write checks canonical names and aliases across all locale, slug uniqueness remains global, and resolver fail-closed rejects any legacy hash that targets several tags. Merge chain resolver has depth/visited guards. Synonyms are explicit pair-unique and expanded one bounded hop, so no recursive loop/explosion.
- Privacy/cache: personal names, descriptions, UUIDs, counts, assignments, deleted rows, selected state and owner controls are absent from guest HTML, public API/search/suggestions/directory/filter/recommendations/structured data/sitemap/shared keys. Personal responses are `private, no-store`; owner-scoped version bump cannot poison a public tag key. Web authoring does not infer personal content language from interface locale. Public cache dimensions are server allowlisted, require the exact canonical scheme/authority, include origin/projection versions, and visibility/moderation invalidation happens after commit; a forged host/port request can never seed shared absolute URLs.
- Import/SSRF: provider URLs are canonicalized to HTTPS `seasonvar.ru`/`www.seasonvar.ru`, reject credentials/non-443 ports/control/bad encoding/path escape and preserve only identity query keys. Raw provider labels remain provenance; new mappings are pending, rejected decisions persist, complete-set convergence removes only orphaned provider assignments. No remote call is added to personal/global tag writes.
- Abuse/concurrency: personal create has account-scoped RateLimiter, per-user and batch limits, owner-local uniqueness and optimistic edit version. API store additionally throttles; idempotent create returns existing exact active tag, assignment reconcile/removal/delete/restore tolerate retries. Global merge/assignment/provider changes use transactions, unique pivots, impact preview/audit and after-commit invalidation.
- Destructive lifecycle: normal global permanent delete is absent; archive/merge preserve URL/mapping/assignment evidence. Personal soft delete has explicit localized action and 30-day restore/conflict rule; expiry purge/account cascade deletes only private tag pivots, never title/watchlist/progress/history/collection/review/comment records.

Owner/profile-media responses are private/no-store; media delivery validates stable owner UUID, allowlisted kind/version and emits `nosniff`.

The current product has no derivative/EXIF pipeline and does not claim one.

## Security внешнего размера и authenticated download

- Metadata inspector и download никогда не принимают URL/extension/filename из request. Source берётся только из persisted authorized `LicensedMedia`; extension — stored allowlisted format, затем URL path/content type без query parameter. Credentials, non-HTTP(S), unexpected port, localhost/`.local`, loopback/private/reserved/link-local/CGNAT/metadata IPv4/IPv6 отклоняются.
- Каждый metadata/download connection повторно resolves все A/AAAA, отклоняет host при любом non-public address и закрепляет validated address через cURL resolve. Download дополнительно требует HTTPS и `PLAYBACK_ALLOWED_HOSTS`. Redirect following отключено, поэтому redirect на новый/unvalidated host никогда не открывается.
- `HEAD`/one-byte Range требуют `Accept-Encoding: identity`; HTML/HLS/encoded representation, malformed Content-Length/Content-Range и ignored Range не дают trusted size. Streamed response закрывается без чтения полного body. Error/event fields содержат только allowlisted category/status/class и redacted URL marker: signed query, cookie, credentials, exception message, body и stack trace не сохраняются.
- Download route требует session auth, scoped title/media relation и current policy/entitlements/health. Denial/mismatch скрывается 404, malformed/multiple/unsatisfied range — 416 без full fallback. Named distributed-cache limiter dimensioned user+IP и user+media ограничивает request starts.
- Upstream headers передаются только allowlisted Content-Length/Range semantics; `Set-Cookie`, Server, CORS/CSP и provider diagnostics не выходят клиенту. Attachment получает sanitized ASCII/RFC 5987 filename, trusted MIME, `nosniff`, `private, no-store`, `noindex`; header/control/path traversal невозможны. Upstream закрывается в `finally`, bytes не персистятся.

Admin route, hydration и каждый action повторно требуют `manage-help-center`; client IDs, enum codes, locale/slug/category/relation/revision/escalation повторно валидируются server-side.

Markdown проходит CommonMark strip и DOM allowlist; raw HTML/script/style/form/iframe/event, JavaScript/data URL, template execution и image удаляются. Route/help links allowlisted, external только HTTP(S) без userinfo/control. Feedback/report используют HMAC actor, CSRF, rate limit, idempotency/dedupe и bounded plain text. Ticket escalation передаёт article UUID только в encrypted expiring Task 20 context. Полный threat/privacy contract: [`help-center.md`](help-center.md).

## Security boundary playback Task 07

Каждый source выбор и signed response повторно проверяют viewer, title/season/episode/media hierarchy, publication/audience/window/health, allowlisted format, HTTPS host/public DNS или safe relative storage path. Client передаёт только opaque media ID; arbitrary URL proxy, path traversal, SSRF, credential forwarding и full-file PHP buffering отсутствуют. Grant TTL ограничен 30–600 секундами, refresh/fallback/preferences/restart rate-limited, responses `private, no-store`, `no-referrer`, `nosniff`, `noindex`.

Report-only CSP `media-src`/`connect-src` ограничен `'self'`, `blob:` и configured `11cdn.org` origins вместо общего `https:`. Provider CORS/Range/MIME остаются deployment contract. Signed URLs не попадают в SEO/shared cache/Livewire public state/storage/report context. Реальных source Premium/region/age полей нет, поэтому bypass и fake client checks не создаются. Полный threat matrix: [`audits/video-playback-report.md`](audits/video-playback-report.md).

## Security и privacy личной библиотеки Task 09

Bookmark, rating, status, feedback/blacklist, progress, marker, acknowledgment и private collection state private by default. `/library/*` использует authenticated owner из session, `account.private` no-store/noindex и policies; user ID, translated status, arbitrary sort column, source URL и collection owner не принимаются. Stable enum/filter allowlists, title→episode ownership, marker duration clamp, CSRF и scoped rate limits закрывают IDOR, mass assignment, duplicate submission и invalid-state paths.

Manual marker хранит только stable episode identity и seconds; optional source/audio/subtitle/provider context намеренно не сохраняется. Personal update определяется server-side published release records и canonical visibility, поэтому client timestamp, technical `updated_at`, hidden/premium/region-inaccessible entry не может создать или снять badge. Public profile/API/SEO/sitemap/shared cache не получают exact status, marker, progress, blacklist или acknowledgment. Public collection presenter сохраняет существующий strict allowlist и не показывает owner private overlay.

Account export содержит semantic codes/timestamps/positions без private media URL; account deletion использует existing cascade/lifecycle. Title merge выполняет server-side precedence и unique reconciliation, поэтому duplicate merge не ослабляет blacklist и не теряет explicit status/marker/update state. Anonymous bookmark/status/blacklist storage не добавлено; существующее anonymous playback storage не расширяется персональными списками.
