# Авторизация

Обновлено: 09.07.2026

## Правила

- Публичные страницы каталога, карточек, sitemap, RSS, OpenSearch и `llms.txt` доступны гостям.
- Операционные и диагностические страницы не должны оставаться публичными.
- `/stats` защищен gate `viewCatalogStats`; сейчас доступ разрешен любому аутентифицированному пользователю.
- Для будущих write/admin/moderation/import-control endpoints нужно добавлять отдельные gates или policies до публикации маршрута.
- Blade-шаблоны не принимают решений авторизации. Допустимы только простые display checks: `@can`, `@cannot`, `@auth`, `@guest`.

## Реализация

- Gate `viewCatalogStats` регистрируется в `App\Providers\AppServiceProvider`.
- Route `/stats` применяет Laravel `can` middleware, поэтому отказ происходит до вызова контроллера.
- Тесты `AuthorizationTest` покрывают гостевой отказ, доступ аутентифицированного пользователя и публичность основных страниц каталога.
