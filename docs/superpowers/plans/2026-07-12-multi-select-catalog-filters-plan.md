# План multi-select фильтров каталога `/titles`

**Дата:** 2026-07-12  
**Ветка:** только существующая `main`; новые ветки, worktree-ветки и PR-ветки не создавать.  
**Режим текущей работы:** код сначала, без создания test-файлов и без запуска PHPUnit/PHP test suite.  
**Основная цель:** дать пользователю возможность выбрать несколько годов, актеров, режиссеров, жанров и других элементов фильтра одновременно, применить их одной GET-формой и сохранить светлый интерфейс без декоративных обводок у текстовых пунктов меню/фильтров.

## Выполненный план текущего прохода

### 1. Сохранить границы проекта

- Работать только в ветке `main`.
- Не создавать и не менять файлы в `tests/`.
- Не запускать `php artisan test`, `./vendor/bin/phpunit` или другие PHP test suites в этом пользовательском режиме.
- Не трогать существующие пользовательские незакоммиченные изменения вне текущей задачи.
- Не добавлять production-зависимости.
- Не менять `.env`.

### 2. Проверить текущий контракт фильтров

- `CatalogTitlesRequest` уже принимает повторяемые массивы:
  - `year[]`;
  - `genre[]`;
  - `country[]`;
  - `actor[]`;
  - `director[]`;
  - `age_rating[]`;
  - `translation[]`;
  - `status[]`;
  - `network[]`;
  - `studio[]`;
  - `tag[]`.
- `CatalogTitlesCriteria` уже хранит годы, relation-slugs, исключения и расширенные фильтры в отдельных полях.
- `CatalogTitlesPageBuilder` уже резолвит выбранные slug-значения пакетно по типу связи, но часть UI и активного состояния оставалась одиночной.
- `CatalogTitleQuery` уже применяет selected/excluded relation IDs, но несколько выбранных IDs одного типа ранее работали как “любой из выбранных”.

### 3. Исправить серверную семантику multi-select

- Годы остаются набором допустимых годов: выбранные `year[]=2023&year[]=2024` показывают тайтлы 2023 или 2024 года.
- Relation-фильтры внутри одной группы требуют совпадения всех выбранных значений:
  - `actor[]=a&actor[]=b` означает тайтлы, где есть актер `a` и актер `b`;
  - `genre[]=drama&genre[]=detective` означает тайтлы, связанные с обоими жанрами;
  - разные группы также объединяются через AND.
- Для этого `CatalogTitleQuery::applyRelationFilters()` использует grouped pivot-подзапрос:
  - `whereIn(related_id, selected_ids)`;
  - `groupBy(catalog_title_id)`;
  - `having count(distinct related_id) = selected_count`.
- ID связей нормализуются перед применением фильтра.

### 4. Сохранить порядок выбранных значений

- `CatalogTitlesPageBuilder` строит `$orderedRecords` в порядке slug-значений из запроса.
- `selectedTaxonomies` и `selectedTaxonomyIds` получают все выбранные записи, а не только первую.
- Активное одиночное значение сохраняется только как совместимость для старых SEO/legacy участков.

### 5. Сделать годы полноценным multi-select

- `CatalogFacetQuery::years()` принимает список выбранных годов.
- Если выбранный год не попал в лимит видимых годов, он добавляется в начало списка, чтобы пользователь мог снять чекбокс.
- Активные годы показываются отдельными chips.
- Удаление одного chip года снимает только этот год, а не все годы сразу.

### 6. Перестроить боковой UI фильтров

- Боковая панель `/titles` стала одной GET-формой.
- Годы и relation-фильтры стали чекбоксами.
- Кнопка «Применить выбранное» применяет сразу несколько выбранных значений.
- Кнопка «Сбросить фильтры» очищает фильтрованный каталог.
- Поиск, сортировка, вид, размер страницы, исключения и расширенные параметры сохраняются скрытыми полями через `CatalogTitlesViewModel::filterFormState()`.
- Строки фильтров не используют outlined-ссылки или декоративную обводку вокруг текста; состояние видно через фон, цвет, жирность и native checkbox.

### 7. Обновить ViewModel

- `CatalogTitlesViewModel::filterFormState()` готовит скрытое состояние формы.
- `CatalogTitlesViewModel::selectedYears()` возвращает нормализованные выбранные годы.
- `CatalogTitlesViewModel::yearQuery()` теперь умеет снять или добавить один год.
- `CatalogTitlesViewModel::isActiveYear()` проверяет список выбранных годов.
- `CatalogTitlesViewModel::isActiveTaxonomy()` проверяет весь список выбранных taxonomies, а не только первый record.
- Расширенные chips больше не дублируют `year`, потому что годы выводятся отдельными chips.

### 8. Защитить SEO/canonical от multi-query ошибок

- `CatalogSeoBuilder::canonicalFromRequest()` теперь умеет сохранять массивы query-параметров.
- Presentation-параметры `sort`, `view`, `per_page` исключаются из fallback canonical.
- Clean taxonomy route используется только для простого одиночного фильтра без поиска и сложного query-state.
- Multi-value комбинации остаются `noindex,follow` через существующую проверку сложного query.

### 9. Синхронизировать документацию

- Обновить `docs/DATA_RELATIONS.md` по поведению `year[]` и relation-групп.
- Обновить `docs/architecture.md` по `CatalogTitlesPageBuilder` и grouped pivot query.
- Обновить `docs/views.md` по `CatalogTitlesViewModel::filterFormState()`.
- Обновить `docs/UI_STANDARDS.md` по GET-форме чекбоксов без outlined-текста.
- Обновить `docs/MAINTENANCE_LOG.md`.
- После ручных правок запустить `php artisan project:docs-refresh`.

### 10. Проверить без PHP test suite

- `php -l` для измененных PHP-файлов.
- `./vendor/bin/pint --dirty --format agent`.
- `php artisan project:docs-refresh`.
- `npm run build`.
- `git diff --check`.
- HTTP smoke на `/titles` с несколькими `year[]`, `genre[]`, `actor[]`.
- Browser smoke на desktop/mobile при наличии Playwright/Chromium. Если браузерный runtime недоступен, минимум проверки — HTTP 200, HTML содержит checked-состояния для выбранных checkbox values, active chips, canonical/robots и кнопку применения.
- Laravel log после smoke не должен получить новых ошибок.
