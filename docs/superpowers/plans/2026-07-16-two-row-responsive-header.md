# План двухполосной адаптивной шапки

**Цель:** Разместить бренд и глобальный поиск в первой полосе, а всю основную навигацию — во второй, сохранив удобство на телефоне, планшете, настольном экране и телевизоре.

**Решение:** Один семантический `header` содержит две независимые области одинаковой максимальной ширины. Верхняя область использует гибкую строку, нижняя — естественный перенос `flex-wrap`. Новое состояние JavaScript, отдельное мобильное меню и внутренняя прокрутка не добавляются.

## Ограничения

- Видимый текст остаётся русским.
- Маршруты, GET-параметр `q`, аудитория ссылок, `aria-current` и Livewire-выход не меняются.
- Цели нажатия остаются не меньше 44 на 44 пикселя.
- На телефоне видны иконки, подписи остаются доступны экранному диктору; с `sm` подписи становятся видимыми.
- Работа выполняется в существующей ветке `main` без включения посторонних изменений.

## Результаты браузерной проверки

- На всех четырёх гостевых и четырёх административных размерах получен HTTP 200; отсутствуют горизонтальное переполнение, ошибки консоли, ошибки страницы и неуспешные запросы.
- Логотип и поиск находятся в одной строке, навигация начинается ниже; на телефоне подписи скрыты визуально, с `sm` видимы.
- Все видимые ссылки, кнопки и поля опубликованной версии имеют фактический размер не меньше 44 на 44 пикселя.
- Общий контракт второй полосы сохраняет этот размер и для кнопок, которые добавляются независимыми функциями после первоначальной верстки.
