# Единый блок фильтров каталога

Дата: 13.07.2026

## Цель

На `/titles` убрать отдельную левую колонку фильтрации и перенести все её группы в существующий раскрывающийся блок «Точный подбор». Каталог должен использовать всю ширину страницы, а фильтры — оставаться доступными на desktop и mobile без отдельного dialog.

## Интерфейс

- «Точный подбор» остаётся одним полноширинным `<details>` между верхней панелью каталога и результатами.
- Блок закрыт по умолчанию и автоматически раскрывается, когда активно хотя бы одно условие фильтрации.
- Счётчик в summary показывает все активные условия, включая годы, справочники, тип публикации, субтитры, диапазоны, видео и качество.
- Внутри сохраняются четыре расширенные группы «Период», «Объём», «Рейтинг» и «Видео».
- Ниже размещаются перенесённые группы: «Годы», «Тип публикации», «Субтитры», «Жанры», «Страны», «Актеры», «Режиссеры», «Возрастной рейтинг», «Озвучка / перевод», «Статус», «Каналы», «Студии» и «Теги».
- На телефоне группы идут в одну колонку, на широком экране — в адаптивные колонки без внутренней прокрутки. Длинные названия и все варианты остаются полностью видимыми.
- Автопоиск актёров и режиссёров, локальный поиск длинных групп, контекстные счётчики и 44-пиксельные touch targets сохраняются.
- Внизу блока находятся действия «Показать результаты» и «Сбросить фильтры». Общий сброс очищает весь набор условий, но не требует отдельного бокового интерфейса.

## Разметка и состояние

- Desktop grid с колонкой `260/280px`, sticky sidebar, native mobile dialog и кнопка открытия dialog удаляются.
- Основная выдача становится единственной полноширинной колонкой.
- Deferred Livewire island фасетов сохраняется: первый ответ не должен вычислять и отправлять все справочники, годы и счётчики.
- Все видимые controls объединяются в одну GET/Livewire-форму. Скрытые поля сохраняют только состояние вне формы, чтобы query keys не дублировались.
- Существующие query-параметры, route-scoped фильтры, `wire:model.live`, GET fallback и поведение browser history не меняются.
- JavaScript диалога удаляется как неиспользуемый. Progressive enhancement поиска по группам и people-combobox остаётся.

## Ошибки и доступность

- Ошибки параметров остаются перед верхней панелью каталога.
- Loading placeholder и live status располагаются внутри единого блока и не блокируют просмотр уже загруженных результатов.
- Summary управляется клавиатурой нативно; отдельная focus-management логика dialog больше не нужна.
- Вложенные scroll-зоны не добавляются. Прокручивается только страница.

## Проверка

- Feature-тест подтверждает отсутствие sidebar/dialog/кнопки открытия и наличие одного `id="catalog-filters"` в «Точном подборе».
- Feature-тест проверяет присутствие перенесённых групп и раскрытие блока при любом активном фильтре.
- Существующие тесты Livewire подтверждают сохранение фильтров, route context и GET fallback.
- Frontend build подтверждает корректность Tailwind-классов и JavaScript после удаления dialog-кода.
- Browser QA проверяет `/titles` на desktop и mobile: ширину выдачи, раскрытие блока, актёрский combobox, отсутствие горизонтального и вложенного scroll.

## Вне области задачи

Query semantics, facet limits, поисковый индекс, сортировка, карточки тайтлов и схема базы данных не изменяются.
