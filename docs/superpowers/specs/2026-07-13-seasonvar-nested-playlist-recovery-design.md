# Восстановление вложенных плейлистов Seasonvar

Дата: 13.07.2026

## Проблема

Страница `serial-7762-Poka_prohodit_zhizn_psfhncj.html` содержит один сезон и 187 серий. Ее JSON-плейлист состоит не из плоского списка элементов с `file`, а из двух групп с полем `folder`. Текущий импортёр рассматривает только верхний уровень, поэтому сохраняет сезоны и серии, но не создаёт `licensed_media`. Публичная карточка показывает только серии с опубликованным источником воспроизведения и в результате выводит 0 доступных серий.

Страница правильно помечается `import_status=missing_data`, однако `retry_after_at` откладывает повторную попытку на 24 часа. При 4 733 текущих страницах `missing_data` полный немедленный повтор при каждом cron-запуске создаст чрезмерную нагрузку.

## Решение

- Рекурсивно обходить элементы Seasonvar-плейлиста и извлекать записи с `file` из `folder` на любой разумной глубине.
- Не ослаблять проверку URL: принимаются только декодированные абсолютные прямые видео-ссылки поддерживаемых форматов.
- Сохранять номер серии из `id`, а при его отсутствии — из заголовка листового элемента.
- В начале каждого обычного и queued цикла выбирать один ограниченный `import.chunk_size` пакет страниц `missing_data`, даже если их `retry_after_at` ещё не наступил. Сортировка по `retry_after_at`, `last_imported_at`, `id` обеспечивает ротацию; существующие lease, Redis-lock и дедупликация в рамках запуска остаются обязательными.
- Обычные HTTP-ошибки продолжают соблюдать backoff и не входят в ускоренный пакет внимания.
- После успешного появления медиа очищать `missing_data_flags` и `retry_after_at` существующей логикой импортёра.

## Проверки

- Feature-тест воспроизводит реальную двухуровневую структуру `folder` и проверяет создание медиа для серий из разных групп.
- Feature-тест проверяет, что свежая страница `missing_data` выбирается при новом запуске до наступления `retry_after_at`, а размер ускоренного пакета ограничен chunk size.
- Существующие тесты плоского плейлиста, повторного импорта, очереди и публичной страницы не должны регрессировать.

## Ограничения

- Новые production dependencies и новые публичные Artisan-команды не добавляются.
- Видео не скачиваются; сохраняются только внешние URL и метаданные.
- Изменение не затрагивает текущие незавершённые UI/Livewire-правки рабочего дерева.
