# Автоматические Livewire islands для фильтров каталога

Дата: 13.07.2026

## Цель

Убрать ручной шаг «Показать фильтры». Карточки каталога должны оставаться в первом быстром HTML, а панель фильтров должна автоматически загружаться сразу после запуска Livewire. Выбранные годы, страны, жанры и остальные значения должны быть визуально отмечены. Изменение checkbox/select должно без отдельной кнопки обновлять выдачу, количество результатов и контекстные счетчики.

## Выбранный подход

Использовать Livewire 4.3 named islands внутри существующего `CatalogSeries`, не создавая дочерний компонент и не добавляя пакеты:

- island результатов рендерится сразу вместе с первым ответом;
- island фильтров использует `defer: true`, поэтому запускается автоматически после загрузки страницы, без клика пользователя;
- оба региона связываются одним именем island, чтобы действие внутри фильтров атомарно перерисовывало фильтры и результаты из одного состояния компонента;
- desktop sidebar остается постоянно раскрытым; на мобильном устройстве автоматически загруженные фильтры остаются внутри существующего native dialog;
- пока deferred island загружается, вместо кнопки показывается компактный недействующий индикатор загрузки.

Связанные named islands выбраны вместо независимых islands, потому что `filters` — общее состояние. Это исключает гонку двух параллельных snapshots, при которой галочка и список карточек могли бы показывать разные значения.

## Данные и производительность

`CatalogTitlesPageBuilder` разделяет подготовку результата и фасетов. Первый SSR вызывает только запросы total/ranked IDs/card hydration и не вычисляет годы, типы публикации, субтитры или taxonomy facets. Deferred island вызывает отдельную facet boundary, которая:

1. строит тот же нормализованный `CatalogTitlesCriteria`;
2. один раз материализует готовые FTS IDs в `CatalogSearchMatchSet`;
3. переиспользует integer-only `json_each(?)` набор для всех contextual facet queries;
4. возвращает render-local collections и `CatalogTitlesViewModel`, не сохраняя модели в публичном Livewire snapshot.

При изменении фильтра связанная группа islands обновляет карточки и новые контекстные счетчики. Сортировка и пагинация используют существующие actions и URL-состояние. Поисковый текст по-прежнему применяется по Enter или «Найти», чтобы не создавать тяжелый запрос на каждый символ.

## Интерфейс

- Удаляются `$facetsLoaded`, `loadFacets()` и разметка кнопки «Показать фильтры».
- Checkbox и select фильтров используют `wire:model.live`; выбранные значения сохраняют активный emerald-фон и нативную отметку `checked`/`selected` после каждого ответа.
- Кнопка «Применить выбранное» больше не требуется для JavaScript-сценария. Обычный GET-submit сохраняется внутри `noscript` как fallback.
- Сброс отдельных групп и полный сброс остаются Livewire actions с обычными `href` fallback.
- Во время обновления показывается scoped loading-состояние, не блокирующее header и остальную страницу.

## Ошибки и ограничения

- Ошибочный или слишком общий поиск продолжает использовать существующие русские сообщения и validation boundary.
- Если deferred запрос завершился ошибкой, уже загруженные карточки остаются доступны; панель показывает нейтральное состояние загрузки Livewire без раскрытия exception text.
- Islands находятся только на верхнем уровне шаблона, не внутри `@foreach`/`@if`, как требует Livewire 4.
- Новых production dependencies, изменения SQLite или внешнего поискового сервиса нет.

## Проверки

- PHPUnit: первоначальный render не выполняет facet SQL; deferred facet boundary возвращает выбранные значения; live-изменение страны сбрасывает страницу и меняет выдачу; сброс восстанавливает состояние.
- Query-plan regression: ranked FTS остается FTS-first и двухфазным; одна загрузка фасетов содержит одну materialization FTS match set.
- Blade/UI tests: присутствуют связанные `@island`, `defer: true` и `wire:model.live`; строка «Показать фильтры» отсутствует.
- Playwright на 390, 768 и 1440 px: фильтры появляются автоматически, выбранная страна отмечена, карточки и count обновляются без reload, dialog/focus работают, horizontal overflow и console errors отсутствуют.
- `./vendor/bin/pint --dirty --format agent`, focused tests, полный `php artisan test`, `npm run build`, `php artisan project:docs-refresh --check`.
