# Полностью русскоязычный технический журнал изменений

Дата: 16 июля 2026 года.

## Цель

Полностью перевести обычный текст `CHANGELOG.md` на русский язык без удаления, объединения, сокращения или смыслового упрощения записей. После перевода новые записи технического журнала также должны добавляться только на русском языке.

## Граница перевода

- Переводится каждый заголовок, подзаголовок, предложение и поясняющий фрагмент обычного текста.
- Сохраняются все даты, числа, измерения, результаты проверок, ограничения, условия отката, сведения о совместимости и перечисления возможностей.
- Точные имена классов, методов, файлов, маршрутов, команд, параметров, переменных окружения, форматов, протоколов, продуктов и сторонних библиотек сохраняются в исходном написании, когда перевод сделал бы техническую ссылку неточной.
- Технические идентификаторы оформляются обратными кавычками там, где это уместно и не меняет существующий смысл.
- Английские общеупотребительные слова внутри русских предложений, например названия действий, состояний, ограничений и этапов проверки, заменяются русскими эквивалентами.
- Аббревиатуры, уже являющиеся точными техническими обозначениями, не раскрываются произвольно и не переименовываются. Смысл таких обозначений объясняется русским окружающим текстом.

## Постоянное правило проекта

В `AGENTS.md` добавляется обязательное правило: обычный текст `CHANGELOG.md` всегда пишется на русском языке. Точные технические идентификаторы разрешены только как часть русского предложения или как фрагменты кода.

Документ `docs/development.md` описывает это правило рядом с существующим русскоязычным контрактом `README.md` и перечисляет локальную команду проверки.

## Автоматическая проверка

Добавляется отдельный сценарий `scripts/check-changelog-policy.sh` со следующими режимами:

- проверка рабочего файла `CHANGELOG.md`;
- проверка версии `CHANGELOG.md`, подготовленной в индексе Git;
- понятная русская ошибка с номером строки;
- пропуск fenced code blocks, адресов, фрагментов в обратных кавычках и точных технических идентификаторов;
- требование кириллицы в каждой содержательной строке обычного текста;
- обнаружение английских грамматических конструкций и цельных английских предложений, даже если строка также содержит отдельные русские слова.

Проверка не пытается переводить или исправлять файл автоматически. Она только отклоняет нарушение и оставляет исходные данные без изменений.

## Интеграция с Git и непрерывной проверкой

- `.githooks/pre-commit` проверяет подготовленную в индексе версию `CHANGELOG.md` вместе с существующей проверкой `README.md`.
- Backend-профиль `scripts/ci-check.sh` проверяет рабочий `CHANGELOG.md`, чтобы правило действовало в GitHub Actions и при локальном полном прогоне.
- Контрактный PHPUnit-тест подтверждает наличие обоих вызовов.

## Разработка через тест

Сначала добавляются тесты, которые должны подтвердить следующие случаи:

1. Полностью русская запись проходит.
2. Точные идентификаторы, команды, маршруты и адреса внутри русского предложения проходят.
3. Полностью английская строка отклоняется с номером строки.
4. Смешанная строка с русским началом и английским предложением отклоняется.
5. Проверка подготовленной в индексе версии читает именно индекс Git, а не рабочий файл.
6. Git-хук и backend-профиль вызывают проверяющий сценарий.

Каждый новый тест сначала запускается в состоянии ожидаемого отказа, после чего добавляется минимальная реализация и выполняется повторный успешный прогон.

## Проверка результата

После полного перевода выполняются:

- проверка отсутствия английской прозы вне разрешённых технических фрагментов;
- проверка количества датированных разделов и записей до и после перевода;
- проверка отсутствия сокращённых, объединённых или удалённых записей;
- новый тест политики журнала;
- контрактные тесты Git-хуков и общего профиля качества;
- `bash -n` для изменённых shell-сценариев;
- `git diff --check`;
- проверка актуальности `README.md` согласно обязательному правилу проекта.

## Критерии готовности

- Весь обычный текст `CHANGELOG.md` написан по-русски.
- Ни одна существующая запись не удалена и не сокращена.
- Точные технические идентификаторы остались корректными.
- Будущая английская строка блокируется до коммита и в GitHub Actions.
- Правило документировано для разработчиков и агентов.
- Все относящиеся к изменению проверки проходят на стабильном состоянии рабочего дерева.
