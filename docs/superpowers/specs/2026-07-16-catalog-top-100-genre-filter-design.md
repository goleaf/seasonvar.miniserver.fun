# Жанровый фильтр рейтингов Top 100 — проектное решение

Дата: 16.07.2026

## Цель

Добавить на четыре публичные страницы `/top/{category}` единый необязательный фильтр по жанру. Посетитель должен выбирать жанр вместе с уже существующими годами и страной, получать заново рассчитанный рейтинг до 100 позиций и сохранять выбранные условия при переходе между фильмами, сериалами, аниме и мультфильмами.

## Выбранный подход

Использовать существующий GET-параметр каталога `genre` со slug жанра. Новые маршруты, таблицы и JavaScript не нужны: `CatalogRecommendationVisibilityService` уже применяет жанровое ограничение через связь `genres`, а `PublicPageCachePolicy` уже включает таксономические параметры каталога в нормализованный cache key.

Жанр становится четвёртым полем immutable `CatalogTopListFilters` после `yearFrom`, `yearTo` и `country`. Фильтр передаётся в общий recommendation context и применяется в исходном SQL до сортировки и `LIMIT 100`.

## Рассмотренные альтернативы

### Исторические снапшоты мест

Они позволяют показать движение тайтлов по дням и неделям, но требуют новой схемы, scheduler, retention policy и backfill. Это отдельный продуктовый этап.

### Персональный Top 100

Персональная выдача полезна зарегистрированным посетителям, но перестаёт быть единым публичным рейтингом и требует приватного cache boundary и объяснимой модели рекомендаций. Эту возможность следует развивать внутри существующего recommendation domain.

### Жанровый фильтр реального каталога

Он дополняет уже опубликованные фильтры, использует существующую таксономию и сразу помогает сузить рейтинг без миграций и новых внешних зависимостей. Выбран этот вариант.

## Архитектурные границы

### Ввод и валидация

`CatalogTopListRequest` нормализует `genre` как необязательное scalar string value и валидирует его:

- длина не больше `CatalogFilterSlug::MAX_LENGTH`;
- значение соответствует каноническому slug через `CatalogFilterSlug`;
- slug существует в `genres.slug` через `Rule::exists(Genre::class, 'slug')`.

Русская и английская локализации получают имя поля и сообщение для неизвестного жанра. Контроллер продолжает работать только с валидированным DTO и не содержит parsing или query logic.

### Состояние фильтров

`CatalogTopListFilters` получает nullable-свойство `genre`. Методы DTO:

- добавляют `genre` в query string только при непустом значении;
- передают `genre` в `CatalogRecommendationContext`;
- считают жанр активным фильтром для reset, empty state и SEO.

Существующий порядок аргументов конструктора сохраняется, жанр добавляется последним, чтобы не ломать текущие вызовы.

### Ранжирование

`CatalogTopListQuery` продолжает передавать `contextFilters()` в `CatalogRecommendationVisibilityService` до category predicates, rating joins, сортировки и `LIMIT 100`. Поэтому места пересчитываются внутри выбранного жанра, а не отсекаются из заранее готовой сотни.

Формула рейтинга не меняется:

- Кинопоиск имеет приоритет, IMDb используется как fallback;
- score сглаживается prior-оценкой `7.0` и `1000` голосами;
- tie-break учитывает число голосов, исходную оценку и ID;
- участвуют только публичные тайтлы с опубликованным доступным media.

### Данные формы

`CatalogTopListFilterOptions` получает bounded-метод `genres()`, который одним запросом выбирает `id`, `name` и `slug`, сортирует по `name`, затем `id`, и ограничивает результат 100 строками. В текущем каталоге 21 жанр, поэтому native select остаётся компактным и предсказуемым.

`CatalogTopListPageBuilder` передаёт Blade:

- выбранный `genre`;
- коллекцию доступных жанров;
- category links с полным нормализованным query state;
- существующие action/reset URL и признак активных фильтров.

Blade не выполняет запросы и не читает `request()`.

## Интерфейс

В существующий блок «Настроить рейтинг» добавляется native select «Жанр» рядом со страной:

- на телефоне элементы идут одной колонкой;
- на `sm` два поля года занимают первую строку, страна и жанр — вторую;
- на широком экране все четыре поля и действия помещаются в одну строку;
- actions занимают обе tablet-колонки и одну desktop-колонку.

Select имеет видимую label, связанное validation message, высоту не меньше 44 px и существующее emerald `focus-visible` состояние. Custom dropdown, modal, inline CSS и JavaScript не добавляются.

При отсутствии результатов используется текущий filtered empty state со ссылкой сброса. При переходе на другую категорию выбранный жанр, годы и страна сохраняются.

## SEO и кеширование

Нефильтрованные Top 100 остаются индексируемыми. Любой query-вариант с жанром использует уже существующий контракт фильтрованных страниц:

- canonical ведёт на базовый URL категории без query string;
- robots получает `noindex,follow`;
- hreflang и JSON-LD `ItemList` не выводятся;
- ссылки на тайтлы остаются обычными публичными ссылками.

`PublicPageCachePolicy` уже разрешает `genre` как параметр таксономии каталога. Производственный код cache policy менять не нужно; characterization test расширяется, чтобы зафиксировать одинаковый ключ для разного порядка параметров и разные ключи для разных жанров.

## Производительность

- жанр применяется через индексированные pivot-отношения внутри основного запроса до `LIMIT 100`;
- список жанров загружается одним bounded-запросом;
- существующая загрузка карточек и eager loading не меняются;
- новая миграция и дополнительная клиентская загрузка не нужны;
- Blade остаётся query-free.

## Ошибки и граничные случаи

- неизвестный или неканонический slug отклоняется validation boundary;
- пустой `genre` нормализуется в отсутствие фильтра;
- существующий жанр без rated/watchable тайтлов показывает filtered empty state;
- category links сохраняют только валидные нормализованные параметры;
- жанр применяется раньше лимита, поэтому подходящий тайтл ниже общей сотни может войти в жанровый рейтинг;
- authenticated viewer state не влияет на публичный порядок.

## Проверки приёмки

- Unit test DTO подтверждает query/context state и active state жанра.
- Feature validation test отклоняет неизвестный жанр с локализованной ошибкой.
- Feature test доказывает применение жанра до `LIMIT 100`.
- Feature tests проверяют выбранное значение, сохранение query в category links и filtered empty state.
- SEO test подтверждает canonical, `noindex` и отсутствие `ItemList` для жанрового варианта.
- Cache test фиксирует нормализацию порядка параметров и разделение разных жанров.
- Browser QA проверяет submit/reset, клавиатурный focus, 390/768/1440/1920 px, отсутствие горизонтального overflow, console и page errors.
- Pint, focused PHPUnit, полный PHPUnit, Vite build, docs check и `git diff --check` завершают этап.

## Вне текущего этапа

- выбор нескольких жанров одновременно;
- индексируемые посадочные страницы жанров;
- autocomplete или custom select;
- историческое хранение мест;
- персональная сортировка;
- изменения формулы рейтинга или новые внешние источники.
