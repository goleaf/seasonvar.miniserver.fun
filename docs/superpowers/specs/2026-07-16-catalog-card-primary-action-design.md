# Удаление избыточного действия карточки каталога

## Цель

Убрать действие «Открыть тайтл» со всех карточек каталога, не ухудшая навигацию и не удаляя полезные персональные действия. Название тайтла и существующая растянутая ссылка карточки остаются основным способом открыть страницу. Для пользователя с историей просмотра продолжают работать «Продолжить просмотр» и «Смотреть снова».

Пользователь утвердил этот вариант формулировкой «ничего не спрашивай, делай всё по твоей рекомендации».

## Проверенный контекст

- Строка «Открыть тайтл» создаётся только в `CatalogUserCardStateLoader::primaryAction()`.
- Общий partial `components.catalog.title-card-personal-state` используется list- и recommendation-карточками, поэтому одно изменение service boundary удалит действие на всех страницах.
- Обе карточки уже содержат ссылку на `titles.show` в названии, растянутую на всю карточку через существующий `after:absolute after:inset-0`.
- Loader записывает `user_in_watchlist=false` даже при отсутствии реального состояния. `TitleCard` сейчас считает само наличие этого атрибута персональным состоянием, поэтому простая замена действия на `null` оставит пустую рамку.

## Выбранная архитектура

`CatalogUserCardStateLoader::primaryAction()` становится nullable boundary:

- без истории просмотра возвращает `null`;
- с незавершённым просмотром возвращает существующее действие `continue`;
- с завершённым просмотром возвращает существующее действие `replay`.

`TitleCard` сначала нормализует четыре видимых персональных значения, затем вычисляет `hasPersonalState` по их содержимому:

- `userInWatchlist === true`;
- `userRating !== null`;
- `userProgressPercent !== null`;
- `userPrimaryAction !== null`.

Само наличие технического атрибута `user_in_watchlist=false` больше не считается видимым состоянием. Blade partial остаётся единым и не получает запросов или бизнес-логики.

## Поток данных

1. Page/query service загружает коллекцию тайтлов.
2. `CatalogUserCardStateLoader` одним набором запросов прикрепляет rating, watchlist и последний meaningful progress.
3. Loader прикрепляет nullable primary action.
4. `TitleCard` нормализует входные значения и определяет, есть ли что выводить.
5. Partial полностью отсутствует в DOM, когда все четыре значения пусты.

Количество запросов и route model binding не меняются.

## Поведение интерфейса

- Ни одна list-, compact- или recommendation-карточка не содержит текста «Открыть тайтл».
- Клик по названию/карточке по-прежнему открывает страницу тайтла.
- «Продолжить просмотр» ведёт к последней серии и `#player`.
- «Смотреть снова» сохраняет прежний маршрут и label.
- Реальная отметка watchlist, rating или progress по-прежнему выводит персональный блок.
- Пустой `data-user-card-state` не выводится.

Новые стили, JavaScript и зависимости не нужны.

## Ошибки и границы

- Некорректный action payload уже нормализуется в `TitleCard::primaryActionAttribute()` в `null` и не выводится.
- Отсутствующий episode/season сохраняет действующую безопасную сборку URL через `array_filter()`.
- Гость не получает пользовательские атрибуты и видит прежнюю карточку без персонального footer.
- Изменение не затрагивает самостоятельные кнопки открытия на странице тайтла, в плеере или в административной части; целевая строка существует только в card-state boundary.

## Проверка

TDD фиксирует:

1. авторизованный пользователь без состояния и progress не видит «Открыть тайтл» и пустой personal-state footer;
2. ссылка названия на `titles.show` сохраняется;
3. watchlist/rating без progress выводятся без общей кнопки открытия;
4. незавершённый progress выводит «Продолжить просмотр»;
5. завершённый progress выводит «Смотреть снова»;
6. число запросов при коллекции карточек остаётся постоянным.

После PHP-изменений выполняются focused PHPUnit tests и Pint. Поскольку Blade-классы и frontend assets не меняются, `npm run build` для этого изолированного изменения не является обязательным; итоговая широкая проверка общего этапа всё равно включает build.

## Документация и выпуск

`README.md` получает посетительскую запись о том, что избыточная кнопка удалена, а переход по карточке и действия продолжения просмотра сохранены. Изменение выпускается независимо от новой версии рекомендаций и не требует миграции или очистки кеша.

## Вне области задачи

- Перепроектирование внешнего вида карточек.
- Изменение текстов «Продолжить просмотр» и «Смотреть снова».
- Добавление новых кнопок или JavaScript-навигации.
- Изменение доступа к страницам тайтлов.
