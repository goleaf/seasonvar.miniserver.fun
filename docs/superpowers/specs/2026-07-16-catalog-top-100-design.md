# Публичные рейтинги «Топ 100»

## Статус и решение

Пользователь делегировал продуктовые и визуальные решения, запретил уточняющие вопросы и попросил провести реализацию от плана до публикации в Git. Спецификация фиксирует четыре самостоятельные публичные страницы:

- `/top/movies` — «Топ 100 фильмов»;
- `/top/series` — «Топ 100 сериалов»;
- `/top/anime` — «Топ 100 аниме»;
- `/top/cartoons` — «Топ 100 мультфильмов».

Один enum, один query-service, один page builder, один controller и один Blade-шаблон обслуживают все категории. Новая логика не дублирует четыре почти одинаковых набора SQL, SEO и разметки.

## Категории каталога

Seasonvar хранит сериал, сезон и эпизоды внутри одного `CatalogTitle`; отдельного надёжного поля «полнометражный фильм» в текущей схеме нет. Поэтому страницы используют явные и проверяемые правила текущего корпуса:

- фильм — публичный неанимационный тайтл типа `serial`, `documentary` или `show` ровно с одним публично доступным эпизодом;
- сериал — публичный неанимационный тайтл тех же типов минимум с двумя публично доступными эпизодами;
- аниме — публичный тайтл с `type=anime`;
- мультфильм — публичный тайтл с жанром `анимационные`, кроме `type=anime`; категория может включать мультсериалы, что прямо объясняется посетителю.

Во всех категориях остаются только тайтлы с опубликованным, доступным и не помеченным сломанным медиа-источником. Личная авторизация не меняет состав и порядок рейтинга; для вошедшего пользователя поверх публичного списка можно загрузить только его состояние карточек.

## Алгоритм рейтинга

Основной источник оценки — Кинопоиск, IMDb используется только при отсутствии оценки Кинопоиска. Простая сортировка по сырой оценке поднимает наверх тайтлы с несколькими голосами, поэтому применяется сглаженная оценка:

`(rating × votes + 7.0 × 1000) / (votes + 1000)`

Она сохраняет понятную десятибалльную шкалу, учитывает доверие к большому числу голосов и остаётся одинаковой для всех категорий. При равенстве используются число голосов, исходная оценка и ID. Страница показывает посетителю исходную оценку, источник и число голосов, но не изображает внутренний score как отдельный пользовательский рейтинг.

Список ограничен первыми 100 позициями. Если доступных оценённых тайтлов меньше, страница показывает честное фактическое количество и не создаёт фиктивные строки.

## UX и визуальная иерархия

- В общей шапке и footer появляется один пункт «Топ 100» с иконкой кубка; четыре категории не раздувают глобальную навигацию.
- Внутри страницы четыре ссылки-вкладки сохраняют текущую локаль и имеют `aria-current`.
- Hero сообщает название, точное назначение списка и краткую методику без рекламного текста.
- Первые три позиции получают отдельный адаптивный «пьедестал» с мягкими различиями золото/серебро/бронза, текстовым номером и той же доступной карточкой каталога.
- Места 4–100 выводятся плотным ordered list: одна колонка на телефоне и планшете, две — на широком desktop/TV.
- Постеры остаются `2:3`, `object-contain`, с честной заглушкой и готовым alt; карточка имеет один основной tab-stop.
- Цвет не является единственным носителем ранга: номер позиции всегда присутствует текстом.
- Touch targets не меньше 44 px; длинные названия, переводы и счётчики переносятся без горизонтального overflow.

## Состояния

- Заполненный список: пьедестал и оставшиеся позиции.
- От одной до трёх записей: только фактически существующие позиции пьедестала.
- Пустой список: точное сообщение о том, что сейчас нет доступных оценённых тайтлов, и рабочая ссылка в общий каталог.
- Авторизованный посетитель: личная оценка, watchlist и продолжение просмотра отображаются существующей карточкой, не влияя на публичный rank.
- Ошибка внешнего источника во время просмотра невозможна: страница не делает внешних HTTP-запросов и читает только локальный каталог.

## SEO и обнаружение

- Каждая категория имеет собственные title, description, canonical и breadcrumbs.
- `CollectionPage` и `ItemList` JSON-LD содержат только фактически выведенные публичные тайтлы и позиции.
- Default RU URL и локализованный EN URL связываются `hreflang`; `/ru/...` остаётся alias канонического default URL.
- Непустые рейтинги входят в streamed static sitemap; пустые тонкие страницы не добавляются.
- URL не принимает фильтры или произвольные query-параметры, поэтому рейтинг остаётся стабильной индексируемой страницей.

## Производительность и безопасность

- Query выбирает только два уникальных provider-rating row на тайтл и использует существующий составной индекс рейтингов.
- Категория ограничивается в SQL до загрузки карточек; relations и counts загружаются bounded grouped queries без N+1.
- Guest HTML использует существующий public page cache домена каталога; authenticated response кеш не разделяет личное состояние.
- Blade не выполняет запросы, не разбирает raw provider data и не содержит inline JavaScript.
- В ответ не попадают external media URLs, source URLs, importer state или закрытые публикации.

## Проверки приёмки

- Feature: четыре URL, enum boundary, категории без пересечений по правилам, watchable/public visibility, fallback IMDb, сглаженное ранжирование, лимит 100.
- View/SEO: вкладки, пьедестал, ordered list, empty state, canonical, breadcrumbs, `CollectionPage` и `ItemList`.
- Navigation/sitemap: один глобальный пункт и четыре непустых canonical URL в static sitemap.
- Quality: Pint, focused PHPUnit, полный test suite, Vite build, `git diff --check`.
- Browser: 375, 768, 1280 и 1920 px, клавиатура, отсутствие horizontal overflow, console/page errors и сломанных изображений.
