# Компактные расширенные фильтры каталога

Дата: 13.07.2026

## Цель

Полностью заменить текущую плоскую сетку «Расширенные фильтры» на компактный и понятный инструмент точного подбора. Пользователь должен быстро понимать, какие параметры относятся к периоду, объёму сериала, рейтингу и видео, не сталкиваясь с растянутыми на всю ширину полями для коротких чисел.

Изменение касается представления существующих фильтров. URL-параметры, правила валидации, `CatalogSeriesFilters`, `CatalogTitlesRequest`, SQL-семантика и обычный GET fallback сохраняются.

## Текущая проблема

Сейчас одиннадцать элементов управления расположены в одинаковой четырёхколоночной сетке. Поля `year_from`, `year_to`, количества сезонов, количества серий, рейтинга и голосов визуально занимают столько же места, сколько длинные текстовые поля, хотя содержат несколько цифр. Между параметрами нет смысловой иерархии, а подписи не объясняют результат фильтрации.

## Информационная архитектура

Блок получает пользовательское название «Точный подбор». Под заголовком выводится короткое описание: «Уточните период, объём сериала, рейтинг и доступность видео».

Параметры делятся на четыре независимые группы:

1. «Период» — год выпуска от/до и давность обновления каталога.
2. «Объём» — количество сезонов от/до и количество серий от/до.
3. «Рейтинг» — источник рейтинга, минимальная оценка и минимальное число голосов.
4. «Видео» — наличие видео и допустимое качество.

Группы не меняют серверную семантику: разные параметры продолжают объединяться через AND, а несколько выбранных качеств — через OR внутри одной группы.

## Компоновка

На ширине `lg` и выше используется сетка 2×2. На телефонах и планшетах группы идут одной колонкой. Каждая группа является компактным `fieldset` со своим `legend`, иконкой, пояснением и внутренней сеткой элементов.

Числовые поля не получают `w-full` на desktop. Для года и рейтинга используется ширина, достаточная для четырёх–пяти символов; для количества сезонов, серий и голосов — ограниченная ширина с адаптивным расширением только на узких экранах. Пары `от / до` располагаются в одной строке, пока ширины хватает, и переносятся без горизонтального скролла.

Минимальная высота интерактивных элементов остаётся 44 пикселя. Компактность достигается шириной, группировкой и вертикальным ритмом, а не уменьшением touch target.

## Группа «Период»

Пояснение: «Ограничьте годы выхода или найдите недавно обновлённые страницы».

- «Год выпуска» — два небольших number input с видимыми маркерами «от» и «до».
- «Обновлено в каталоге» — компактный набор взаимоисключающих вариантов: «Любое время», «За день», «За неделю», «За месяц», «За год».
- Текущее значение `updated` сохраняет существующие URL-значения `day`, `week`, `month`, `year`.

## Группа «Объём»

Пояснение: «Выберите короткий мини-сериал или многосезонную историю».

- Строка «Сезоны» содержит number input «от» и «до».
- Строка «Серии» содержит number input «от» и «до».
- Рядом с парой отображается единица измерения, чтобы значение не воспринималось как произвольное число.
- Существующие границы валидации остаются источником истины.

## Группа «Рейтинг»

Пояснение: «Укажите источник оценки и минимальный уровень доверия».

- «Источник» — компактный select с вариантами «Любой», «КиноПоиск», `IMDb`.
- «Оценка не ниже» — number input от 0 до 10 с шагом 0,1.
- «Голосов не меньше» — ограниченный number input.
- Под полями выводится короткая нейтральная подсказка: высокий рейтинг при малом числе голосов может быть менее показательным.

## Группа «Видео»

Пояснение: «Показывайте страницы с доступным видео и нужным качеством».

- Наличие видео остаётся select со значениями «Неважно», «Есть видео», «Нет видео».
- Качества `2160p`, `1440p`, `1080p`, `720p`, `480p`, `360p`, `240p` отображаются компактными checkbox controls с явным selected-состоянием.
- Качества не становятся декоративными кнопками: checkbox остаётся доступным для клавиатуры и screen reader.

## Состояния и действия

`details` автоматически раскрывается, когда активен хотя бы один расширенный параметр. В summary показывается число активных расширенных параметров; при отсутствии выбора счётчик не создаёт визуального шума.

Внизу формы располагаются:

- основная кнопка «Показать результаты»;
- вторичное действие «Сбросить точный подбор», очищающее только расширенные поля и качества;
- существующий общий сброс каталога остаётся отдельным действием на странице.

Во время `applyFilters` кнопка блокируется и показывает существующее Livewire loading-состояние. Ошибки диапазонов продолжают выводиться в общей доступной панели ошибок над каталогом; поля сохраняют введённые значения.

## Data flow и границы кода

- Blade остаётся presentation-only: без `@php`, запросов, cache/service calls и подготовки массивов.
- Значения и URL-state продолжают принадлежать `CatalogSeriesFilters` и `CatalogTitlesViewModel`.
- Если для summary или сброса группы нужны новые вычисления, они добавляются типизированными методами в `CatalogTitlesViewModel` или Form Object, а не inline-выражениями в Blade.
- Отправка использует существующий `wire:submit="applyFilters"` и GET action `titles.index`.
- Выбранные базовые taxonomy/year/search/sort/view параметры сохраняются скрытыми полями, как сейчас.
- Новые зависимости и JavaScript для блока не добавляются.

## Доступность и адаптивность

- Каждая группа использует `fieldset`/`legend`.
- Подсказки связываются с controls через `aria-describedby`, когда относятся к конкретному набору.
- Focus-visible остаётся двухпиксельным emerald-контуром проекта.
- На ширине 320 пикселей пары полей переносятся и не вызывают горизонтальный скролл.
- Цвет не является единственным признаком выбранного качества: сохраняются checkbox и текст.
- Summary остаётся keyboard-operable native control.

## Тестирование

Feature/UI tests должны подтвердить:

- наличие четырёх смысловых `fieldset`;
- сохранение всех существующих `name`, `wire:model` и GET-параметров;
- автоматическое раскрытие при активном расширенном фильтре;
- отображение числа активных расширенных параметров;
- наличие отдельного сброса только точного подбора;
- отсутствие `@php`, PHP tags и query/cache calls в Blade;
- сохранение 44-пиксельных touch targets;
- отсутствие `w-full` у коротких number controls на desktop;
- корректную работу диапазонной валидации и Livewire reset action.

Browser QA выполняется для mobile, tablet и desktop: проверяются перенос пар, отсутствие overflow, читаемость пояснений, loading-state и сохранение состояния после применения.

## Критерии приёмки

- Блок воспринимается как четыре понятных шага точного подбора, а не как набор одинаковых полей.
- Короткие значения вводятся в визуально пропорциональные controls.
- Пользователь понимает назначение каждого параметра без внешней инструкции.
- Все текущие фильтры и URL остаются обратно совместимыми.
- Search, taxonomy filters, сортировка, пагинация и обычный GET fallback не меняют поведение.
- Интерфейс остаётся русскоязычным, светлым, адаптивным и доступным.
