# Глобальный поиск в шапке и единый русский редакторский режим

## Статус и решение

Спецификация заменяет прежний контракт текстового Livewire-autocomplete. Пользователь разрешил не задавать дополнительные вопросы и применять рекомендуемую архитектуру.

Выбран двухскоростной публичный поиск:

1. быстрый контур сериалов возвращает богатые карточки с постером, годом, количеством доступных сезонов и серий;
2. независимый контур портала ищет остальные публичные сущности и не может задержать карточки сериалов;
3. обычная GET-форма остаётся резервным сценарием без JavaScript;
4. полный переход из dropdown ведёт на общую страницу поиска, а не притворяется бесконечным списком внутри шапки.

Граница данных: поиск видит только публичные сериалы, публичные значения справочников, публичные подборки, публичные заявки, публичные профили и зарегистрированные публичные разделы. Личные, скрытые, удалённые, ожидающие модерации и административные записи не попадают ни в ответ, ни в кеш.

## Рассмотренные варианты

### Один синхронный Livewire-запрос

Минимально меняет текущий код, но связывает скорость сериалов с самым медленным источником. Фактический аудит показал, что быстрый FTS тайтлов блокируется запросом тегов на 500–700 мс, а пользователь видит результат через 1,2–2 секунды. Вариант отклонён.

### Единая новая таблица поисковых документов

Даёт один быстрый запрос и удобное ранжирование, но требует миграции, полного backfill, наблюдателей или очередей для каждого домена и отдельного восстановления индекса. Для текущего объёма и грязного рабочего дерева это слишком широкая граница. Вариант оставлен как будущий этап при доказанном росте p95.

### Два независимых API-контура

Переиспользует существующий read-only `/api/v1/search/suggestions`, позволяет отменять устаревшие запросы и показывает сериалы сразу после их ответа. Специализированные bounded queries заменяют тяжёлые page-query сервисы. Этот вариант выбран.

## UX-контракт шапки

- Поиск начинается после двух нормализованных букв или цифр.
- Debounce равен 150–180 мс; каждый новый ввод отменяет оба предыдущих запроса через `AbortController`.
- Клиент хранит маленький session-memory cache по `scope + locale + normalized query`.
- Контур `header_titles` и контур `header_portal` запускаются параллельно.
- Сериалы появляются сразу после готовности первого контура; портал дополняет уже открытую панель позднее.
- Устаревший ответ не может заменить результаты нового запроса.
- Ошибка одного контура не скрывает успешно загруженный второй контур.
- Состояния загрузки, пустого ответа и ошибки доступны через `aria-live`.
- Обычная `<form method="GET">` остаётся рабочей; без JavaScript она ведёт на общую страницу поиска.
- `ArrowUp`, `ArrowDown`, `Home`, `End`, `Enter`, `Escape`, Tab и клик вне компонента имеют детерминированное поведение.
- Панель не содержит внутренней прокрутки. Она растёт вместе со страницей и использует ограниченную header-выдачу.
- Поле поиска не получает цветную рамку, `ring` или `outline` при hover, focus, вводе или загрузке. Постоянная нейтральная рамка закреплена в `docs/UI_STANDARDS.md`.

## Карточка сериала

Каждый результат типа `title` содержит:

- безопасный web URL;
- публичное название;
- оригинальное название, только когда оно отличается;
- `poster_url` или честную светлую заглушку;
- год либо отсутствие года без вымышленного значения;
- количество доступных текущему публичному посетителю сезонов;
- количество доступных текущему публичному посетителю серий.

Постер имеет фиксированное отношение `2:3`, `object-contain`, `referrerpolicy="no-referrer"`, не растягивается и не обрезается на телефоне, планшете, desktop или TV. Все счётчики вычисляются grouped queries после получения bounded списка ID; relation queries из Blade/JavaScript отсутствуют.

## Публичные источники портала

Контур `header_portal` может возвращать совпадения следующих типов:

- жанры, страны, актёры, режиссёры;
- возрастные ограничения, переводы каталога, статусы;
- телеканалы/сети, студии, публичные тематические теги, годы;
- публичные редакционные и пользовательские подборки;
- публичные заявки на материалы;
- публичные активные профили;
- зарегистрированные публичные разделы портала.

Результаты группируются для представления как «Люди», «Справочники», «Сообщество» и «Разделы». Внутренний `type` остаётся точным. Ранжирование внутри общего portal-контура: точное совпадение, начало строки, начало слова, вхождение, затем стабильный приоритет источника и название.

Комментарии, отзывы, технические обращения, личные теги, email, биографии, описания, исходные URL, private notes и raw importer state не индексируются и не ищутся.

## API-контракт

Существующий endpoint `/api/v1/search/suggestions` сохраняет legacy-ответ без обязательного `scope` для совместимости мобильных клиентов.

Новые allowlisted значения:

- `scope=header_titles` — только богатые публичные карточки сериалов;
- `scope=header_portal` — только остальные публичные сущности.

Неизвестный scope даёт JSON validation error. Опциональная локаль ограничена конфигурацией интерфейса и входит в URL/cache key. Ответ остаётся Laravel API Resource и содержит только allowlisted поля. Для title добавляются nullable `url`, `poster_url`, `original_title`, `year`, `seasons_count`, `episodes_count` и готовая локализованная `meta`. Для portal item добавляются `url`, `group` и `meta`.

Header-scopes всегда используют публичную аудиторию и не персонализируются по session/token. Это сохраняет безопасный общий кеш и исключает утечку entitlement/private данных.

## Производительность

- Тяжёлый `TagQuery::searchPublic()` не используется в header-пути.
- Для тегов существует отдельный lightweight query без загрузки translations/aliases/synonyms и без нескольких вложенных `withCount`.
- Каждый источник выбирает только поля, нужные ответу, и включает обязательные PK/FK для отношений.
- Лимиты применяются внутри источника до объединения.
- Счётчики тайтлов выполняются двумя grouped queries по bounded ID, без N+1.
- API/public cache и доменный suggestion cache разделяются по scope, locale и SHA-256 нормализованного запроса.
- Header response не содержит описаний и длинного пользовательского текста.
- Цель после прогрева: server p95 до 150 мс для `header_titles` и до 250 мс для `header_portal`; cold response до 350 мс на текущей SQLite-базе. Цель интерфейса — первая полезная карточка до 450 мс при нормальной сети.

## Общая страница поиска

`GET /search?q=...` является канонической точкой полного поиска по порталу:

- сервер валидирует и нормализует `q`;
- показывает богатую группу сериалов и публичные portal-группы;
- предлагает явный переход в `/titles?q=...` для полной пагинации сериалов;
- ссылки справочников, подборок и заявок ведут в существующие канонические страницы;
- страница не индексируется (`noindex,follow`) и не попадает в sitemap;
- пустой запрос показывает форму, а не запускает broad scan.

Dropdown остаётся bounded preview. «Безлимитность» реализуется маршрутом результатов и переходами к пагинируемым доменным страницам, а не тысячами DOM-строк в шапке.

## Адаптивность

- Контрольные ширины: 375, 390, 768, 1280, 1440 и 1920 px.
- Touch targets — минимум 44×44 px.
- На телефоне панель занимает доступную ширину viewport с безопасными отступами; постер не меньше 48×72 px.
- На tablet/desktop/TV ширина панели ограничена читаемым `max-width`; строки не растягиваются на весь TV.
- Длинные названия и метаданные переносятся, не перекрывая постер или кнопку.
- Нет горизонтального переполнения и внутренних scroll-зон.
- TV/keyboard управление не зависит от hover.

## Язык интерфейса и редакторский язык

### Язык интерфейса

- RU/EN-переключатель полностью удаляется из общей шапки и других публичных мест.
- Изменять язык интерфейса может только авторизованный пользователь в профиле, раздел «Внешний вид».
- Старый POST endpoint быстрого переключения удаляется, если после аудита не имеет других обязательных клиентов.
- Прямые локализованные публичные URL продолжают работать; существующие English translation files и данные не удаляются.

### Русский редакторский режим

- Конфигурация получает единый `authoring_locale=ru` для подборок и тегов.
- Новая редакционная подборка создаётся с `content_locale=ru`, независимо от языка интерфейса.
- Редактор подборки всегда открывает и сохраняет русскую translation row; RU/EN-кнопки и надпись «Русский» отсутствуют.
- Личный тег создаётся и сохраняется с русским `content_locale`; языковой select и подпись отсутствуют.
- Админка тегов показывает одну форму русского текста без заголовка языка; новый alias сохраняется как `ru` без visible selector.
- Существующие английские переводы/alias rows остаются в базе и public read paths, но текущие формы их не удаляют и не перезаписывают.
- `content_locale`, original/audio/subtitle language у заявок и технических обращений остаются: это метаданные произведения/медиа, а не переключатель редакторского интерфейса.

## Безопасность и отказоустойчивость

- Ввод: 2–80 нормализованных символов, без raw SQL fragments и wildcard amplification.
- URLs строятся только именованными локальными routes; клиент отбрасывает ответ с другим origin.
- Постеры используют только готовый публичный URL и никогда не загружаются сервером ради подсказки.
- API не раскрывает private/hidden/admin records, email, importer state, source URL или stack trace.
- Abort/stale-response logic исключает перерисовку старым запросом.
- При отказе API остаётся обычная GET-форма.

## Проверки приёмки

- Unit: ранжирование, лимиты, лёгкий запрос тегов, публичные границы каждого источника, grouped counts, cache scope/locale.
- Feature/API: validation scopes, backward compatibility legacy API, JSON shape, no private records, `/search` noindex.
- Blade/contract: нет Livewire query loop, нет цветного focus frame, poster/count markup и GET fallback.
- JavaScript unit/contract: AbortController, stale responses, безопасное DOM-создание, keyboard navigation.
- Playwright: картинки/заглушки, год/сезоны/серии, progressive arrival, error isolation, keyboard/touch, 375/768/1280/1920, отсутствие overflow и console/network errors.
- Language: locale control только в account settings; collection/tag authoring не содержит RU/EN/«Русский» и сохраняет `ru`; media language fields остаются.
- Final gates: Pint, focused tests, full PHPUnit, Vite build, docs refresh/check, diff check и README actuality.
