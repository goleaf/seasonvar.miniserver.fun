# Фильтры рейтингов Top 100 — проектное решение

Дата: 16.07.2026

## Цель

Расширить четыре публичных рейтинга `/top/{category}` серверными фильтрами по диапазону годов и стране производства. Фильтры должны работать одинаково для фильмов, сериалов, аниме и мультфильмов, применяться до расчёта мест и сохранять текущие требования к доступности, SEO, кешированию и скорости.

## Выбранный подход

Использовать существующие GET-параметры каталога:

- `year_from` — необязательный первый год диапазона;
- `year_to` — необязательный последний год диапазона;
- `country` — необязательный slug страны.

Новые route variants, JavaScript и таблицы базы данных не нужны. `CatalogRecommendationVisibilityService` уже умеет ограничивать публичные тайтлы по этим полям, а `PublicPageCachePolicy` уже включает их в catalog cache key. Такой подход сохраняет одну каноническую структуру URL и не создаёт вторую систему фильтрации.

## Рассмотренные альтернативы

### Исторические снапшоты и динамика мест

Преимущество — посетитель видит изменения рейтинга по дням и неделям. Недостаток — необходимы новая схема, scheduler, политика хранения и backfill. Это отдельный продуктовый этап и не требуется для полезной фильтрации текущего рейтинга.

### Персональный Top 100

Преимущество — индивидуальная выдача. Недостаток — рейтинг перестаёт быть общим публичным списком, требует приватного cache boundary и объяснимой модели персонализации. Эта возможность должна развиваться внутри существующего recommendation domain.

### Серверные фильтры по реальным полям каталога

Преимущество — немедленная пользовательская ценность без миграции и внешних зависимостей. Выбран этот вариант.

## Архитектурные границы

### Ввод

`CatalogTopListRequest` нормализует scalar query values и валидирует:

- годы как целые числа от `1900` до следующего календарного года;
- страну как существующий `countries.slug`, прошедший общий `CatalogFilterSlug` с пределом 120 символов;
- порядок диапазона: `year_from` не больше `year_to`.

Все сообщения ошибок локализованы. Контроллер получает только уже проверенный request и не содержит правил фильтрации.

### Состояние фильтров

Immutable `CatalogTopListFilters` хранит `yearFrom`, `yearTo` и `country`. DTO предоставляет:

- фильтры для `CatalogRecommendationContext`;
- очищенный query array для ссылок и формы;
- признак активных фильтров.

Blade не читает `request()` и не нормализует query string.

### Ранжирование

`CatalogTopListQuery` принимает DTO и передаёт его значения в существующий visibility boundary до category predicates, rating joins, сортировки и `LIMIT 100`. Поэтому место означает позицию внутри выбранного периода и страны, а не фильтрацию уже готовой сотни.

Алгоритм рейтинга не меняется:

- Кинопоиск имеет приоритет, IMDb используется как fallback;
- score сглаживается prior-оценкой `7.0` и `1000` голосами;
- tie-break: число голосов, исходная оценка, ID;
- только публичные тайтлы с опубликованным доступным media.

Список стран готовит отдельный query-free для Blade сервис из таблицы `countries`, ограниченный 100 строками и отсортированный по названию. В текущем каталоге 67 стран, поэтому native select остаётся bounded.

### Page builder

`CatalogTopListPageBuilder` готовит:

- action URL формы и reset URL;
- значения полей, список стран и признак активных фильтров;
- category links с сохранённым валидным query state;
- разные empty-state тексты для пустой категории и пустого результата фильтрации;
- podium, основной ordered list, count и SEO contract.

Все URL формируются route helpers в PHP.

## Интерфейс

Под category navigation появляется один компактный светлый блок «Настроить рейтинг»:

- два числовых поля «Год от» и «Год до»;
- native select «Страна»;
- основная кнопка «Показать рейтинг»;
- плоская ссылка «Сбросить», видимая только при активных фильтрах.

На телефоне controls идут одной колонкой, с `sm` годы объединяются, на широком экране форма занимает одну строку. Каждый control имеет видимую label, `min-height: 44px`, связанное сообщение ошибки и emerald `focus-visible`. Внутренний scroll, custom dropdown, modal, inline CSS и JavaScript отсутствуют.

При пустом фильтрованном результате страница объясняет, что выбранным условиям ничего не соответствует, и предлагает сбросить фильтры. Исходная пустая категория сохраняет переход в полный каталог.

## SEO и кеширование

Базовые страницы без фильтров сохраняют `index,follow`, canonical, hreflang, `CollectionPage` и `ItemList`.

Любая фильтрованная страница:

- использует canonical базовой страницы категории без query string;
- получает `noindex,follow`;
- не выводит hreflang и JSON-LD списка, чтобы не создавать индексируемые дубликаты;
- сохраняет обычные внутренние ссылки на тайтлы.

Catalog page cache разделяет варианты по нормализованным `year_from`, `year_to` и `country`. Новая cache policy не вводится.

## Производительность

- фильтры применяются внутри основного SQL до `LIMIT 100`;
- `catalog_titles.year` и обе стороны `catalog_title_country` уже индексированы;
- страны загружаются одним bounded запросом;
- карточки по-прежнему загружаются существующим bounded loader без N+1;
- новая миграция не требуется.

## Ошибки и граничные случаи

- неизвестная страна отклоняется validation boundary;
- перевёрнутый диапазон показывает локализованную ошибку;
- один край диапазона разрешён;
- страна без подходящих rated/watchable тайтлов даёт честный filtered empty state;
- category links сохраняют только валидные нормализованные значения;
- authenticated viewer state не меняет публичный порядок.

## Проверки приёмки

- Feature tests доказывают применение страны и диапазона до ранжирования.
- Feature tests покрывают сохранение query в category links, reset URL и filtered empty state.
- Validation tests покрывают неизвестную страну, границы лет и перевёрнутый диапазон.
- SEO tests подтверждают base canonical и `noindex`/отсутствие ItemList для query-варианта.
- Cache test подтверждает разные ключи для разных фильтров.
- Browser QA проверяет submit/reset, keyboard focus, 390/768/1440/1920 px, отсутствие horizontal overflow и console/page errors.
- Pint, focused PHPUnit, полный PHPUnit, Vite build, docs check и `git diff --check` завершают этап.

## Вне текущего этапа

- историческое хранение мест;
- персональная сортировка;
- отдельные landing pages для каждой страны или года;
- client-side ranking и autocomplete страны;
- новые внешние источники рейтингов.
