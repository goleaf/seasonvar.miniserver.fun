# Надёжная пагинация и раздельные алфавиты

Дата: 14.07.2026

## Цель

Сделать переключение страниц надёжным во всех Livewire-списках портала, после успешного переключения плавно возвращать пользователя к началу обновлённой выдачи и разделить кириллические и латинские буквенные фильтры в каждом интерфейсе, где алфавит уже поддерживается.

## Подтверждённая причина

Production-проверка указанного маршрута `/titles/country/turciia?country[0]=turciia` показала, что исправный Livewire-запрос меняет URL, активную страницу и карточки. Общая реализация остаётся хрупкой по двум причинам:

- `resources/views/vendor/livewire/tailwind.blade.php` выводит активные элементы пагинации только как `button` с `wire:click`, без реального `href`; без работающего Livewire у пользователя нет обычного GET fallback;
- `resources/js/app.js` ищет только `[data-catalog-results]`, поэтому существующий подъём работает на `/titles`, но не имеет явной цели в directory hubs, `/watching` и `/admin/catalog`.

Алфавитный UI также расходится с уже описанным проектным контрактом: `/actors` и `/directors` смешивают обе письменности в одном списке, а `/titles` показывает кириллицу и одну агрегированную кнопку `A–Z` вместо отдельных латинских букв.

## Область изменения

Пагинация:

- `/titles`, `/titles/year/{year}` и `/titles/{type}/{taxonomy}`;
- все одиннадцать directory hubs, когда их результат занимает несколько страниц;
- история `/watching`;
- административный каталог `/admin/catalog`;
- общий Livewire pagination override и его frontend-scroll contract.

Алфавит:

- desktop и mobile controls основной выдачи `/titles`;
- `/actors`;
- `/directors`.

JSON API сохраняет существующие pagination metadata и не получает browser-scroll поведения. Статические Laravel paginator links уже являются обычными ссылками и не требуют изменения семантики.

## Решение по пагинации

### Прогрессивное улучшение

Активные элементы общего Livewire pagination override становятся ссылками с реальными URL:

- `Назад` использует `previousPageUrl()`;
- номер страницы использует URL из paginator elements;
- `Вперёд` использует `nextPageUrl()`;
- `wire:click.prevent` сохраняет быстрый Livewire-переход, когда JavaScript доступен;
- без JavaScript браузер выполняет обычный GET по `href`.

Disabled и current-page состояния остаются некликабельными `span`. Русские подписи, `aria-current`, `rel="prev|next"`, минимальная высота 44 px, светлая тема и стабильные `wire:key` сохраняются.

### Явная цель прокрутки

Каждый вызов `links()` передаёт собственный `scrollTo` selector в общий pagination view. Цель находится на начале именно той выдачи, которую обновляет paginator:

- каталог тайтлов — `[data-catalog-results]`;
- directory hub — `[data-directory-results]`;
- история — `[data-viewing-history-results]`;
- административный каталог — `[data-admin-catalog-results]`.

Pagination control публикует selector через `data-pagination-scroll-to`. Frontend сохраняет его при активации, а после успешного Livewire morph вызывает существующий `smoothAnchorScroll()`. Так прокрутка относится к уже обновлённой выдаче, а не к старому DOM перед запросом. Функция продолжает учитывать `scroll-margin-top`, sticky header и `prefers-reduced-motion`. При ошибке Livewire автоматический подъём не выполняется.

Обычный GET fallback начинает новую загрузку в стандартной позиции страницы. URL query, route context, выбранные фильтры, сортировка, вид и размер страницы сохраняются paginator-ом и существующими page builders.

## Решение по алфавиту

### Основной каталог

Desktop и mobile получают одинаковые смысловые группы:

- `Кириллица` — существующий кириллический ряд каталога;
- `Латиница` — отдельные `A, B, C, …, Z`;
- `#` — отдельный нейтральный фильтр названий, начинающихся не с буквы;
- `Все` — сброс текущего буквенного фильтра.

Каждая буква использует существующий query key `letter`, Livewire action `setLetter()` и обычный GET URL. Новый client state не вводится.

Старый `letter=latin` остаётся допустимым и продолжает означать всю латиницу для обратной совместимости существующих URL. Новый основной UI создаёт ссылки на отдельные латинские буквы, а пользовательское представление legacy-значения использует подпись `Латиница A–Z`, а не техническое слово `latin`.

### Справочники актёров и режиссёров

Сервисная граница группирует только реально доступные первые буквы опубликованных значений:

- кириллица выводится в кириллическом порядке;
- латиница выводится от `A` до `Z`;
- `#` остаётся отдельным control;
- пустая группа не рендерится.

Фильтрация, `letter` URL state, browser history и сброс paginator остаются в `CatalogDirectoryBrowser`. Blade получает уже подготовленные группы и не классифицирует буквы самостоятельно.

## Данные и архитектура

Порядок и классификацию букв хранит небольшой presentation-safe PHP-класс в текущей каталожной границе. Его используют `CatalogTitlesViewModel` и `CatalogDirectoryPageBuilder`, чтобы desktop, mobile и directory UI не расходились. Класс не обращается к базе и возвращает простые массивы букв.

`CatalogDirectoryQuery` по-прежнему отвечает только за получение доступных initials. Page builder преобразует эту коллекцию в готовые группы. Никакие запросы не переносятся в Blade.

Схема базы, route names, query keys, API-форма и importer не меняются. Новые production dependencies не добавляются.

## Ошибки и граничные случаи

- Невалидный `letter` продолжает нормализоваться или отклоняться существующей валидацией.
- Пустая алфавитная группа не оставляет пустой заголовок.
- Out-of-range page продолжает канонизироваться существующим component redirect.
- Во время Livewire-запроса ссылки получают `inert`, поэтому быстрые повторные клики не запускают конкурирующие переходы.
- Если scroll selector отсутствует или больше не существует после morph, JavaScript безопасно завершает обработку без ошибки.
- Reduced-motion пользователь получает мгновенный подъём без анимации.

## Проверки

PHPUnit покрывает:

- наличие реального `href` и Livewire action у `Назад`, номеров и `Вперёд`;
- сохранение route-backed country и query filters в URL второй страницы;
- переключение paginator state и изменение набора результатов;
- четыре явные scroll targets Livewire-списков;
- отдельные кириллические и латинские группы `/titles`;
- полный ряд `A`–`Z` и legacy `letter=latin`;
- раздельные, правильно упорядоченные доступные группы `/actors` и `/directors`;
- отсутствие запросов и inline PHP в Blade.

Playwright на `1440×1200` и `390×844` проверяет:

- указанный country route с количеством карточек больше одной страницы;
- изменение URL, active page и первой карточки после перехода;
- положение viewport у начала обновлённой выдачи;
- работу `Назад`;
- раздельные видимые группы кириллицы и латиницы;
- отсутствие горизонтального переполнения, console/page errors и failed local assets.

После реализации запускаются focused PHPUnit tests, `./vendor/bin/pint --dirty --format agent`, полный `php artisan test`, `npm run build` и релевантный Playwright suite.

Устойчивый контракт обновляется в `docs/catalog-search.md`, `docs/frontend.md`, `docs/views.md`, `docs/UI_STANDARDS.md` и `CHANGELOG.md`. Управляемые блоки `project-docs` вручную не редактируются.

## Не входит в задачу

- редизайн карточек, фильтров или шапки;
- infinite scroll;
- изменение размера страниц;
- изменение API pagination schema;
- добавление алфавита в справочники, где буквенной фильтрации сейчас нет;
- изменение поискового индекса или importer parsing.
