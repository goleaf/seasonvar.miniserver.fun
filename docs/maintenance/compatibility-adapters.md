## Explicit non-adapters

- `CatalogWatchStatus` values (`planned`, `watching`, `paused`, `completed`, `dropped`) and recommendation feedback values are canonical persisted identities, not translated aliases.
- `Route::get(..., LivewireComponent::class)` is a supported route registration used consistently by the existing application. It is not presented as a deprecated compatibility shim merely because `Route::livewire()` is now preferred.
- No old class-name alias, duplicate event-name bridge, service-worker cache version or payment/OAuth SDK adapter was found. Their absence is recorded instead of inventing migration work.

## Removal procedure

Removal requires repository-wide code/config/provider/route/middleware/migration/asset/storage/cache/environment/deployment/documentation search, production dependency evidence, migration and rollback records, affected-feature verification and an update to this file. Unknown dependants mean `retain`.
