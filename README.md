# Seasonvar — каталог сериалов

Seasonvar — русскоязычный каталог сериалов, передач, аниме и документальных проектов. Портал объединяет карточки произведений, сезоны, серии, подборки, рекомендации, отзывы, обсуждения и безопасные ссылки на внешнее видео.

Проект активно развивается. Этот файл содержит краткий обзор возможностей, инструкции запуска, состояние работ, дорожную карту и понятную посетителю историю обновлений.

## Состояние проекта

- Основной сайт работает на Laravel 13.20, PHP 8.5 и Livewire 4.3.
- Интерфейс собирается через Tailwind CSS 4.3 и Vite 8; Plyr и HLS.js используются для воспроизведения.
- Локальная разработка и автоматические тесты используют SQLite; очереди, кеш и фоновые задачи могут использовать Redis и Memcached.
- Единственная публичная команда импорта Seasonvar — `php artisan seasonvar:import`.
- Основная рабочая ветка Git — `main`.
- Обычный текст этого файла всегда должен оставаться на русском языке. Точные команды, маршруты, имена технологий, файлов и переменных сохраняются в исходном написании.

## Последние новости

Актуальность раздела проверена 16 июля 2026 года.

- Раздел `/collections` теперь автоматически получает редакционные подборки из разрешённого источника: портал сохраняет локальные WebP-обложки, включает только уверенно найденные в собственном каталоге произведения и использует общую подборку как отдельную понятную причину сходства.
- Открыты четыре постоянных рейтинга: «Топ 100 фильмов», «Топ 100 сериалов», «Топ 100 аниме» и «Топ 100 мультфильмов». В них попадают только доступные для просмотра материалы, места рассчитываются по оценке и числу голосов без влияния личной истории посетителя, а выдачу можно уточнить по диапазону лет и стране.
- Добавлен полный фоновый прогрев всех безопасных общедоступных страниц каталога: он охватывает каждую опубликованную карточку, страницы справочников и пагинации, но принципиально не затрагивает профили, личные данные, вход, просмотр и скачивание видео.
- Публичные профили больше не раскрывают заголовок или фрагмент рецензии, если автор отметил её как спойлер; сообщения об ошибочных параметрах списка рецензий теперь учитывают выбранный язык интерфейса.
- Главная страница снова корректно показывает новое видео, добавленное к ранее опубликованной серии.
- Каталог, поиск, подборки, теги и фоновые обновления теперь запрашивают у базы только действительно используемые поля связанных записей. Это уменьшает объём передаваемых и удерживаемых данных без сокращения содержимого страниц.
- В поиске верхней полосы появилось быстрое автодополнение: карточки сериалов с постером, годом и грамматически верным числом сезонов и серий выводятся независимо от более широкого поиска по справочникам, людям, публичным подборкам, заявкам, профилям и разделам. Подписи совпадают с языком текущей страницы, поиск выполняется только по названиям, повторные запросы берутся из кеша и сохраняется обычная кнопка «Найти» как резервный вариант. Рамка поля всегда остаётся нейтральной, без зелёной обводки при вводе.
- Добавлена общая страница результатов `/search`: она объединяет найденные сериалы и безопасные публичные разделы портала, не показывает скрытые материалы и позволяет перейти к полной выдаче каталога.
- В меню зарегистрированного пользователя снова доступна отдельная ссылка «Профиль», а логотип получил понятное доступное имя даже в компактном мобильном виде.
- Автоматические проверки GitHub и локальная проверка перед отправкой теперь используют один сценарий для PHP, тестов, документации, сборки и браузерной матрицы. Исправлена несовместимая команда форматирования, из-за которой прежние запуски GitHub Actions завершались ошибкой.
- Верхнее меню разделено на две адаптивные полосы: логотип и поиск всегда находятся сверху, а навигация — ниже. На телефоне используются компактные иконки, на более широких экранах появляются подписи, поэтому пункты больше не мешают поиску.
- Открыт раздел заявок на недостающие сериалы, сезоны, серии, переводы, субтитры, улучшение качества и исправление информации. Пользователь может голосовать, подписываться на изменения и следить за состоянием своей заявки.
- Добавлены персональные и общедоступные рекомендации: популярное, тренды, высокие оценки, новинки, недавние обновления, будущие релизы, редакционные подборки и похожие произведения.
- Зарегистрированные пользователи могут безопасно скачивать поддерживаемые прямые видеофайлы. Видео передаётся потоком и не сохраняется на сервере портала.
- На карточке серии показывается известный размер прямого видеофайла. Старые записи постепенно получают эти сведения в фоновом режиме.
- Усилена защита входа, восстановления пароля, сессий и мобильных токенов без изменения привычных страниц авторизации.
- Улучшена скорость общедоступных подборок, рейтингов и проверки доступности серий.
- Добавлен отдельный закрытый раздел технических обращений для ошибок сайта, проигрывателя и уже опубликованных материалов.
- Главная страница полностью встроена в русскую и английскую локализацию. Выбор языка доступен только в настройках профиля; публичный переключатель и языковые переключатели редакторов удалены, а материалы подборок и тегов теперь создаются в едином русском режиме без лишней подписи языка.

## Возможности для посетителей

### Каталог и поиск

- Полный каталог по адресу `/titles` поддерживает поиск, сортировку, постраничную навигацию и сохранение состояния в адресной строке.
- Поле поиска в шапке после двух символов показывает сгруппированные подсказки по названиям сериалов, справочников, людей, публичных подборок, заявок, профилей и разделов портала. Стрелки, Enter и Escape работают с клавиатуры; без JavaScript остаётся обычный переход на общую страницу результатов.
- Фильтры работают по годам, жанрам, странам, актёрам, режиссёрам, переводам, студиям, сетям, возрастным ограничениям, статусам, качеству видео и наличию субтитров.
- Раздел «Точный подбор» позволяет ограничить период выхода, продолжительность сериала, рейтинг и свойства видео.
- Отдельные справочники доступны для жанров, стран, актёров, режиссёров, переводов, студий, сетей, тегов, статусов, возрастных ограничений и годов.
- Старые и изменённые адреса карточек перенаправляются на актуальные страницы без раскрытия скрытых материалов.

### Карточка сериала и просмотр

- Карточка объединяет описание, постер, рейтинги, связи, сезоны, серии, переводы, субтитры, трейлеры и доступные варианты видео.
- Проигрыватель поддерживает обычные файлы и HLS, выбор доступного варианта, переход к предыдущей или следующей серии и понятные сообщения об ошибках.
- Внешний адрес видео не помещается в страницу: портал выдаёт короткоживущую внутреннюю ссылку и повторно проверяет доступ перед воспроизведением.
- Прогресс просмотра сохраняется для вошедшего пользователя и используется в разделах «Продолжить просмотр» и «История».
- Прямые файлы поддерживают безопасное скачивание с продолжением по диапазону байтов. HLS остаётся только для просмотра через интернет.

### Учётная запись и личная библиотека

- Доступны регистрация, вход, подтверждение почты, восстановление пароля, выход и управление активными сессиями и мобильными устройствами.
- В настройках можно выбрать язык, часовой пояс, параметры проигрывателя, скорость, громкость, качество, субтитры, горячие клавиши и уменьшение анимации.
- Личная библиотека объединяет список просмотра, оценки, состояния сериалов, продолжение просмотра, историю и скрытые рекомендации.
- Пользователь может создавать приватные, доступные по ссылке и общедоступные коллекции, менять порядок произведений и загружать обложки.
- Личные теги помогают организовать библиотеку и не становятся общедоступными без явного действия пользователя.
- У каждого пользователя есть постоянное публичное имя и отдельный профиль: владелец сам открывает описание, коллекции, опубликованные отзывы, комментарии и безопасные списки «Смотрю»/«Просмотрено». Подробная история и точный прогресс всегда остаются закрытыми.
- В профиле можно загрузить аватар и обложку, изменить имя пользователя с подтверждением паролем, настроить приватность разделов, заблокировать или приглушить другого пользователя и отправить жалобу модератору.
- Экспорт и удаление учётной записи учитывают связанные личные данные и не раскрывают сведения других пользователей.

### Рекомендации и подборки

- Общедоступные подборки находятся по адресу `/discover/{type}`.
- Именованные пользовательские и редакционные коллекции с обложками и полным составом находятся по адресу `/collections`; автоматически обновляемые редакционные карточки отмечены отдельным бейджем.
- В разделе «Топ 100» доступны отдельные страницы фильмов, сериалов, аниме и мультфильмов: `/top/movies`, `/top/series`, `/top/anime` и `/top/cartoons`.
- На каждой странице «Топ 100» можно выбрать начальный и конечный год выпуска и страну; выбранные условия сохраняются при переходе между четырьмя категориями.
- Рейтинг использует оценку КиноПоиска или запасную оценку IMDb вместе с числом голосов; статистическая поправка не позволяет единичной высокой оценке вытеснить устойчиво оценённые произведения.
- Персональная выдача учитывает реальный прогресс, список просмотра, состояния, оценки, собственные коллекции и личные теги.
- Если личных данных ещё недостаточно, портал честно показывает общедоступную подборку без выдуманного процента совпадения.
- Продолжения, предыстории, ответвления и связанные произведения показываются отдельно от рассчитанного сходства.
- Действие «Не интересно» можно отменить в личной библиотеке.

### Заявки, отзывы и общение

- Раздел `/requests` принимает заявки на отсутствующие или требующие исправления материалы и повторно проверяет возможные совпадения перед созданием.
- Обсуждения доступны для сериала, сезона, серии и коллекции; ответы, реакции, жалобы и ограничения проходят единые правила доступа.
- Пользователь может оставить одну структурированную рецензию на сериал, отметить спойлер, изменить её, удалить и восстановить в разрешённый срок.
- Спойлерный текст не попадает в начальную страницу, уведомления, поисковые данные и служебные описания до явного раскрытия.
- Личные сообщения, обращения, жалобы и служебные заметки не попадают в общедоступный кеш.

### Технические обращения

- Ошибки уже существующего контента, проигрывателя и функций портала принимаются отдельно от заявок на новый контент и жалоб на пользователей.
- Пользователь после входа создаёт обращение на `/issues/new`, отслеживает его на `/issues`, отвечает на уточнения и проверяет результат исправления; подтверждение почты не блокирует сообщение о неисправном письме подтверждения.
- Случайный номер `ISS-…` служит только безопасной ссылкой для общения: просмотр и каждое действие по-прежнему требуют серверной авторизации.
- Необязательные диагностика и снимки экрана очищаются от чувствительных данных, хранятся приватно и не попадают в карту сайта или общий кеш.
- Полный технический договор находится в [`docs/technical-issues.md`](docs/technical-issues.md).

### Мобильный программный интерфейс

- Стабильный интерфейс `/api/v1` предоставляет главную, каталог, фильтры, справочники, карточки, сезоны, серии, рекомендации и отзывы.
- Мобильная авторизация использует ограниченные по сроку токены Sanctum с отдельными правами чтения и изменения.
- Личные методы возвращают список просмотра, оценки, состояния, продолжение просмотра, историю и краткую сводку библиотеки.
- Синхронизация поддерживает контрольные точки, последовательную загрузку изменений и идемпотентную пакетную отправку.
- Воспроизведение использует короткоживущие разрешения и не раскрывает сохранённые внешние адреса видео.
- Полный технический договор находится в [`docs/api.md`](docs/api.md).

## Импорт Seasonvar

Обычный последовательный запуск:

```bash
php artisan seasonvar:import
```

Обновление одной разрешённой страницы с повторным разбором:

```bash
php artisan seasonvar:import "https://seasonvar.ru/serial-615--Bez_sleda_pssmtlk-1-season.html" --force
```

Дополнительные режимы:

```bash
php artisan seasonvar:import --inventory-only
php artisan seasonvar:import --forever
php artisan seasonvar:import --queued
php artisan seasonvar:import --status
php artisan seasonvar:import --refresh-media-sizes
```

- Режим `--inventory-only` сверяет разрешённые адреса карты сайта и не изменяет каталог.
- Режим `--forever` предназначен для одного долгоживущего процесса.
- Режим `--queued` ставит работу в Redis и используется с настроенными обработчиками очереди.
- Режим `--refresh-media-sizes` небольшими пакетами обновляет сведения о размере прямых файлов.
- Подробная установка служб, расписаний, обработчиков и порядок безопасного переключения находятся в [`docs/deployment.md`](docs/deployment.md).

Импортёр не скачивает и не хранит полные видеофайлы. Он сохраняет внешние адреса, качество, формат, перевод, субтитры, состояние доступности и известный размер. Для проверки допускаются только заголовок `HEAD` или минимальный диапазон `Range: bytes=0-0`.

## Синхронизация редакционных подборок

Источник отключён по умолчанию. Сначала выполните полный просмотр без записи:

```bash
HDREZKA_COLLECTION_SYNC_ENABLED=1 php artisan catalog-collections:sync-hdrezka --dry-run
```

После проверки счётчиков запустите синхронизацию:

```bash
HDREZKA_COLLECTION_SYNC_ENABLED=1 php artisan catalog-collections:sync-hdrezka
```

Команда обходит индекс и всю найденную пагинацию, но добавляет в состав только уверенно сопоставленные записи уже существующего локального каталога. Неоднозначные и ненайденные названия остаются в диагностике и не создают фиктивные фильмы. Загружаются только обложки самих подборок: они ограничиваются по размеру, преобразуются в WebP и сохраняются на локальном приватном диске; видео и постеры отдельных произведений этот процесс не скачивает.

После материального изменения затронутые рекомендации точечно перестраиваются в Redis-очереди, а критические страницы передаются существующему прогреву Redis/Memcached. Ежедневное расписание по умолчанию запускается в `03:37`; лимиты, задержка, очередь и обработка обложек задаются переменными `HDREZKA_COLLECTION_*` из `.env.example`.
